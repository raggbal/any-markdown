# v0.195.396 - iframe方式サイドパネル + リンク開きモード設定

## 課題

1. 前回(v0.195.394)のサイドパネルはdiv+contenteditable方式で、markdownToHtmlFragmentでHTMLを生成してinnerHTMLにセットしていたため、コードブロック/Mermaid/Math/Undo/Redo等のエディタ機能が動作しなかった
2. mdリンクの開き方を設定で切り替えたい（サイドパネル/別タブ）
3. Cmd+Click（Ctrl+Click）で常に別タブで開きたい
4. サイドパネル内でmdリンクをクリックした場合の動作が未考慮だった

## 原因

- div+contenteditable方式では、editor.jsの初期化処理（コードブロックUI、Mermaid描画、UndoManager、キーハンドラ等）が適用されない
- editor.jsは13000行の単一IIFEで、メインエディタのDOM要素にハードバインドされており、2つ目のインスタンスを作れない

## 対策

### 設計

**iframe方式**: サイドパネル内に`<iframe>`で完全に独立したエディタインスタンスを埋め込む。

```
メインwebview (editor.js)
├── メインエディタ (通常通り)
├── サイドパネル枠 (header + iframe container + overlay)
│   └── <iframe src="blob:...">
│       ├── side-panel-host-bridge.js (parent.postMessage経由)
│       ├── editor.js (完全なエディタインスタンス)
│       ├── vendor/ (同じwebview URIで外部読み込み)
│       └── styles.css + data-side-panel="true" (sidebar/side-panel非表示)
└── メッセージ中継ハンドラ (iframe ↔ VSCode)
```

**メッセージフロー:**
- iframe → parent.postMessage({source:'sidePanel', ...}) → editor.js → host.xxx() → VSCode
- VSCode → postMessage → editor.js → iframe.contentWindow.postMessage → iframe

### 実装

1. **package.json**: `fractal.linkOpenMode`設定追加（`"sidePanel"` / `"tab"`、デフォルト: `"sidePanel"`）

2. **host-bridge.ts / vscode-host-bridge.js / test-host-bridge.js**:
   - `openLinkInTab(href)`: Cmd+Click用、常に別タブで開く
   - `sidePanelOpenLink(href, sidePanelFilePath)`: サイドパネル内リンク用、サイドパネルファイルからの相対パス解決

3. **side-panel-host-bridge.js（新規）**: iframe内で動作するHostBridge。全メソッドが`parent.postMessage({source:'sidePanel', ...})`経由でメインwebviewに中継。`reportEditingState`/`reportFocus`/`reportBlur`等は空実装。

4. **webviewContent.ts**: `getSidePanelHtml()`関数追加。メインHTMLと同じvendor URI/styles/editor.jsを使うが、HostBridgeはside-panel-host-bridge.js、ツールバーはsimple固定、CSS追加で`.sidebar`と`.side-panel`を非表示。blob URL iframe用に`frame-src blob:`をメインCSPに追加。

5. **editorProvider.ts**:
   - `openLink`/`openLinkInTab`ハンドラ統合: `linkOpenMode`設定を読み、`tab`なら`vscode.openWith`、`sidePanel`なら`getSidePanelHtml()`でHTML生成して`openSidePanel`メッセージ送信
   - `sidePanelOpenLink`ハンドラ: サイドパネル内リンクを受信し、サイドパネルファイルの親ディレクトリ基準で相対パス解決。再度`getSidePanelHtml()`で新しいパネルHTMLを送信（サイドパネル内ナビゲーション）

6. **editor.js**:
   - リンクclick: `e.metaKey || e.ctrlKey`でCmd+Click判定→`host.openLinkInTab()`
   - サイドパネル: div+contenteditable方式をiframe方式に完全書き換え
   - `openSidePanel(html, filePath, fileName)`: Blobオブジェクト→`URL.createObjectURL()`→iframe.src
   - iframe→parentメッセージリスナー: `edit`→`host.saveSidePanelFile()`, `openLink`→`host.sidePanelOpenLink()`

7. **editor-body-html.js**: `<div class="side-panel-editor" contenteditable>`を`<div class="side-panel-iframe-container" id="sidePanelIframeContainer">`に変更

8. **styles.css**: `.side-panel-body`/`.side-panel-editor`を`.side-panel-iframe-container`に統合（`flex:1; overflow:hidden`）
