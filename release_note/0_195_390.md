# v0.195.390 - サイドバーフッターから「ライブプレビューモード」表示を削除

## 課題

- サイドバーフッターに「ライブプレビューモード」/「Source Mode」のステータス行が表示されていた
- この表示はユーザーにとって冗長であり、サイドバーの縦スペースを無駄に消費していた
- v0.195.389で画像ディレクトリ情報をサイドバーフッターに追加したため、縦スペースの節約がより重要になった

## 原因

- `webviewContent.ts` に `<div class="sidebar-status-mode" id="statusLeft">` が定義されていた
- `editor.js` でソースモード切替時に `statusLeft.textContent` を更新するロジックがあった
- `styles.css` に `.sidebar-status-mode` のCSSルールがあった

## 対策

### 設計

1. サイドバーフッターからモード表示行を完全に削除
2. ソースモード切替時のステータス更新ロジックを除去
3. 関連するCSSルールを削除

### 実装

#### `webviewContent.ts`
- `<div class="sidebar-status-mode" id="statusLeft">` 要素を削除

#### `editor.js`
- `const statusLeft = document.getElementById('statusLeft');` を削除
- ソースモード切替ハンドラから `statusLeft.textContent = i18n.sourceMode;` と `statusLeft.textContent = i18n.livePreviewMode;` を削除

#### `styles.css`
- `.sidebar-footer .sidebar-status-mode { margin-top: 4px; }` ルールを削除

## 変更ファイル

- `src/webviewContent.ts`
- `src/webview/editor.js`
- `src/webview/styles.css`
