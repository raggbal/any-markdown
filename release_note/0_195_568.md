# v0.195.568 リリースノート

## バグ修正: 画像ペースト時のファイル名重複

### 課題
v0.195.567 で追加した Outliner ノード画像機能で、2枚目以降の画像を Cmd+V で貼り付けても、常に1枚目の画像が表示される。

### 原因
クリップボードからの画像ペースト時、ブラウザが返す `file.name` は常に同じデフォルト名（例: `image.png`）。これをそのままホストに渡していたため:
1. ホスト側で同じファイル名で上書き保存される
2. ブラウザが同一 URL をキャッシュし、最初に読み込んだ画像を表示し続ける

### 対策
- ペースト時にファイル名を `null` で送信するよう変更
- ホスト側で `image_${Date.now()}.${ext}` の一意なタイムスタンプベースファイル名を自動生成

## 設定削除: `fractal.outlinerForceRelativeImagePath`

### 課題
`fractal.outlinerForceRelativeImagePath` 設定が `package.json` に定義されていたが、コード上どこからも参照されていなかった。

### 対策
- Outliner の画像パスは常に `.out` ファイルからの相対パスで保存されるため、この設定は不要
- `package.json` から削除
