# v0.195.494 — Outliner: ↑↓ナビゲーション・初回フォーカス・スコープ/サイドパネル閉じ時フォーカス

## 課題1: 検索結果での↑↓矢印キーが別の行に飛ぶ

### 原因

検索ツリーモードでは、マッチノード＋祖先＋子孫がすべて`currentSearchResult` Setに入る。`renderNodes`がツリーを再帰的に描画するため、**DOMに同じ`data-id`を持つ要素が複数存在**する（例: 142ノード中、多数のIDが重複）。

以前の実装:
1. `model.getPreviousVisibleId()` / `model.getNextVisibleId()`: モデルのDFS走査で次IDを返す。検索でフィルタされたDOMには存在しないノードIDを返す場合がある。
2. `focusNode(nodeId)`: `querySelector('.outliner-node[data-id="xxx"]')` で最初のマッチにフォーカス。重複IDがあると上方の祖先コンテキスト側に飛ぶ。

### 対策

IDベースのルックアップを廃止し、**DOM要素の同一性比較（`===`）**に切り替え。

- `getDomPrevNodeEl(currentTextEl)` / `getDomNextNodeEl(currentTextEl)`: 現在の`.outliner-text`から`.closest('.outliner-node')`で親ノード要素を取得。`querySelectorAll`全ノードと`===`比較で現在位置を特定し、前後のDOM要素を直接返す。
- `focusNodeEl(nodeEl)` / `focusNodeElAtStart(nodeEl)`: DOM要素を直接受け取りフォーカス。`querySelector`によるID検索を回避。
- 矢印キーハンドラ: `getDomNextNodeEl(textEl)` → `focusNodeEl(nextEl)` のチェーンに変更。

## 課題2: outliner初回オープン時に1行目にフォーカスが入らない

### 原因

以前は`loadData`メッセージハンドラで`focusNode(model.rootIds[0])`を呼んでいたが、webviewのレンダリング完了前にフォーカスを試みていた。また初期化コード（`init()`）にはフォーカスロジックがなかった。

### 対策

- `loadData`ハンドラを削除（初期化は`init()`で完結）
- `init()`で非空ツリーの場合、`setTimeout(100)` で `focusFirstVisibleNode()` を呼び出し
- `focusFirstVisibleNode()`: `treeEl.querySelector('.outliner-node')` → `focusNodeElAtStart()` でDOM要素直接方式

## 課題3: スコープ変更・サイドパネル閉じ時にフォーカスが消える

### 原因

`setScope()`や`closeSidePanelImmediate()`で`renderTree()`後にフォーカスを戻す処理がなかった。

### 対策

- `setScope()`: `renderTree()`後に`focusFirstVisibleNode()`を呼び出し
- `closeSidePanelImmediate()`: クリーンアップ後に`focusFirstVisibleNode()`を呼び出し

## 課題4: `focusNode` / `focusNodeAtStart` で `focusedNodeId` が更新されない

### 原因

`focusNode(nodeId)`は`textEl.focus()`するが`setFocusedNode(nodeId)`を呼んでおらず、内部状態とDOMフォーカスが不一致になる場合があった。

### 対策

`focusNode()` / `focusNodeAtStart()` の両方に `setFocusedNode(nodeId)` を追加。
