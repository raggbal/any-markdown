# v0.195.500 — ページ解除時に .md ファイルをゴミ箱に移動

## 課題

Outlinerで「ページ解除」を実行すると、ノードの `isPage` / `pageId` はクリアされるが、対応する `.md` ファイルがディスクに残り続け、孤立ファイルが蓄積する問題があった。

また、`outlinerProvider.ts` に `removePage` メッセージのハンドラが存在せず、ホスト側でページ解除を検知できない状態だった。

## 原因

1. **pageId の消失**: `model.removePage()` が `pageId` を `null` に設定した後にホストへメッセージを送信していたため、ホスト側でファイルパスを特定できなかった
2. **ハンドラ未実装**: `outlinerProvider.ts` に `removePage` の case が存在しなかった

## 対策

### 設計

- `model.removePage()` が旧 `pageId` を返すように変更し、ホストブリッジ経由でプロバイダに渡す
- `vscode.workspace.fs.delete()` の `useTrash: true` オプションでOS のゴミ箱に移動（誤操作時の復元が可能）
- サイドパネルで該当ページが開かれている場合は、削除前に閉じる

### 実装（変更ファイル）

| ファイル | 変更内容 |
|---------|---------|
| `src/webview/outliner-model.js` | `removePage()` が旧 `pageId` を返すように変更 |
| `src/webview/outliner.js` | 返り値の `pageId` を `host.removePage()` に渡す |
| `src/shared/outliner-host-bridge.js` | `removePage(nodeId, pageId)` に引数追加 |
| `src/outlinerProvider.ts` | `handleRemovePage` メソッド追加（ゴミ箱移動 + サイドパネル閉じ） |

### 処理フロー

```
ユーザー右クリック → 「ページ解除」
  ↓
outliner.js: removePage(nodeId)
  ↓ saveSnapshot() → model.removePage(nodeId) → pageId取得
  ↓ host.removePage(nodeId, pageId)
  ↓
outliner-host-bridge.js: { type: 'removePage', nodeId, pageId }
  ↓
outlinerProvider.ts: handleRemovePage()
  ↓ サイドパネルで開いていたら閉じる
  ↓ vscode.workspace.fs.delete(uri, { useTrash: true })
```
