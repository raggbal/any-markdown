# v0.195.410 - Electron対応: サイドパネル + Action Panel

## 課題

4da1671（v0.195.393）以降に追加された2つの主要機能がVSCode拡張でのみ動作し、Electronデスクトップアプリでは未対応だった。

1. **サイドパネル（iframe方式）**: `.md` リンクをクリックしてNotion風サイドパネルで開く機能
2. **Action Panel（ページ追加）**: Cmd+Nでフローティングパネルを表示し、新規ページ作成・リンク挿入する機能

editor.js側のUI/ロジックはプラットフォーム非依存で完成済みだったが、Electron側にIPCハンドラとHTML生成関数がなかった。

## 原因

HostBridge抽象化パターン（editor.js → window.hostBridge → ホスト固有実装）において、Electron側の実装（preload.ts/main.ts）に新メソッドが追加されていなかった。

具体的に不足していた要素:
- `preload.ts`: 8つのIPC送信メソッド（サイドパネル4 + Action Panel4）
- `html-generator.ts`: サイドパネル用の自己完結HTML生成関数
- `main.ts`: 対応するIPCハンドラ群、サイドパネルファイルwatcher管理
- `file-manager.ts`: ファイル検索用ユーティリティ

## 対策

### 設計

既存のElectronアーキテクチャパターンに完全に準拠:
- `preload.ts` の `contextBridge.exposeInMainWorld` パターンでメソッド追加
- `main.ts` の `ipcMain.on()` パターンでハンドラ追加
- `file-manager.ts` の chokidar パターンでサイドパネルファイル監視

サイドパネルHTML生成の最重要制約: blob: URLからはfile://リソースを読み込めないため、vendor JS/CSS（turndown, mermaid, katex等 計3.5MB）を全てインライン埋め込み。

### 実装

#### 変更ファイル（4ファイル）

**electron/src/preload.ts**
- 8メソッド追加: `openLinkInTab`, `saveSidePanelFile`, `sidePanelOpenLink`, `notifySidePanelClosed`, `searchFiles`, `createPageAtPath`, `createPageAuto`, `updatePageH1`

**electron/src/html-generator.ts**
- CSPに `frame-src blob:;` と `script-src blob:` 追加
- `generateSidePanelHtml()`: vendor全インライン、side-panel-host-bridge.js注入、toolbar=simple固定
- `extractToc()`: Markdownからh1/h2見出し抽出（editorProvider.tsからの移植）

**electron/src/file-manager.ts**
- `static searchMdFiles()`: 再帰ディレクトリウォークによる.mdファイル検索（`.`始まりとnode_modules除外）

**electron/src/main.ts**
- `open-link` ハンドラ拡張: HTTP→外部ブラウザ、#→スクロール、.md→サイドパネル、その他→shell.openPath
- サイドパネルwatcher管理: `setupSidePanelWatcher()` / `disposeSidePanelWatcher()`
- 8新規IPCハンドラ: サイドパネル操作（保存/ナビゲーション/クローズ）+ Action Panel操作（検索/作成/H1更新）
- ウィンドウクローズ時のwatcherクリーンアップ
