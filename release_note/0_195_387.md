# v0.195.387 リリースノート

## 変更内容: Electronアプリ自動アップデート通知 & GitHub Actionsリリース自動化

### 課題
- Electronデスクトップアプリにアップデート通知機能がなく、ユーザーが新バージョンの存在を知る手段がなかった
- ビルド＆リリースが手動のみで、macOS/Windows/Linuxの全プラットフォームをビルドするにはそれぞれのマシンが必要だった
- VSCode拡張とElectronアプリのバージョンが独立管理（0.195.386 vs 0.1.0）で不整合があった

### 対策

#### 設計
- コード署名なし（$0）で実現できる軽量な通知方式を採用
  - GitHub Releases API (`/releases` 全件) から `electron-v` プレフィックスのタグを検索
  - VSCode拡張のリリースと競合しない（`/releases/latest` ではなく全件検索）
  - 新バージョン検知時はダイアログ通知 → ブラウザでReleasesページを開く（手動DL）
- GitHub Actions で `electron-v*` タグ push 時に4プラットフォーム自動ビルド
- バージョンは VSCode拡張と Electron で常に同一値を維持

#### 実装

**新規ファイル:**
- `electron/src/updater.ts`: GitHub API でバージョンチェック、ダイアログ通知、Releasesページ開く
  - 起動15秒後に初回チェック、以降24時間ごと自動チェック
  - 手動チェック（メニュー）は結果を必ず表示
  - 外部パッケージ依存なし（Electron組み込みの `net` モジュール使用）
- `.github/workflows/release-electron.yml`: 4ジョブ並列ビルド
  - macOS arm64 (macos-latest), macOS x64 (macos-13), Windows (windows-latest), Linux (ubuntu-latest)
  - softprops/action-gh-release で自動アップロード

**修正ファイル:**
- `electron/src/main.ts`: `setupUpdateChecker()` 呼び出し追加
- `electron/src/menu.ts`: Help メニューに「Check for Updates...」追加
- `electron/package.json`:
  - version を 0.195.387 に統一
  - Linux ターゲット (AppImage, deb) 追加
  - macOS ターゲットに x64/arm64 両アーキテクチャ指定
- `.claude/skills/build/SKILL.md`: Electronビルド手順追加
- `.claude/skills/publish/SKILL.md`: Electron公開手順追加（タグpush→Actions自動ビルド）
- `docs/electron-auto-update.md`: 実装ガイドドキュメント作成
