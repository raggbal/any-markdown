# v0.195.332 - Perplexityテーマをライト系に修正

## 課題

v0.195.331で追加したPerplexityテーマが、実際のPerplexity.aiの回答画面と大きく異なっていた。
- テーマの背景色が真っ黒（`#0C0C0C`）になっていた
- 実際のPerplexity.aiは白〜ウォームホワイト系の背景を使用している
- アクセントカラーもサーモンピンク（`#F5C1A9`）で、実際のターコイズ系（`#1FB8CD`）と異なっていた

## 原因

ブランドガイドラインのダーク系カラーパレットを誤って参考にした。Perplexity.aiの実際の画面デザイン（ライト背景 + ターコイズアクセント）ではなく、ブランドカラーのダーク系バリエーションを採用してしまった。

## 対策

### 設計

[Perplexity公式ブランドカラー](https://live.standards.site/perplexity/color)を参照し、以下のカラーパレットを採用：

| 色名 | Hex | 用途 |
|---|---|---|
| Paper White | `#FBFAF4` | 背景色 |
| Offblack | `#13343B` | テキスト色 |
| Turquoise 800 | `#0B363C` | 見出し色 |
| True Turquoise | `#1FB8CD` | リンク色・アクセント |
| Light Gray | `#EAEEEF` | コード背景 |
| Turquoise 100 | `#DEF7F9` | ボーダー・選択色 |
| Teal | `#2E565E` | 引用テキスト色 |
| Terra Cotta | `#A84B2F` | シンタックスハイライト（数値） |
| Olive | `#707C36` | シンタックスハイライト（文字列） |
| Warm Red | `#BF505C` | エラー表示色 |

### 実装

以下のファイルを修正：

**src/webview/styles.css**
1. CSS変数をダーク系→ライト系に全面変更（12変数すべて）
2. シンタックスハイライト色をライト背景に適した色に変更（13色すべて）
3. スタイルオーバーライドのリンクアンダーライン色をターコイズ系に変更
4. HR（水平線）のボーダー色をライト系に変更
5. テーブルのヘッダー背景・ボーダー色を追加
6. コードブロックにボーダーを追加
7. Mermaid/Mathエラー表示色をライト系に変更（背景: `#FFF0F0`、文字: `#BF505C`）
