# v0.195.389 - 画像ディレクトリ設定UIの改善

## 課題

- Simpleツールバーモードでは `imageDir` ボタンが非表示になり、画像保存先を設定する手段がなくなっていた
- VSCode設定やデフォルトの画像保存先パスが何処なのか、ユーザーに表示されていなかった
- `currentImageDir` が未設定（ディレクティブなし）の場合、サイドバーフッターには何も表示されなかった

## 原因

- ツールバーの `imageDir` ボタンは `insert` グループに属しており、Simpleモードでは `insert` グループ全体が `display: none` で非表示
- サイドバーフッターの `statusImageDir` は `currentImageDir`（ファイル内ディレクティブ）がある時のみ表示する設計だった
- VSCode設定の `imageDefaultDir` やデフォルト（ドキュメントと同じディレクトリ）の情報はWebviewに送信されていなかった

## 対策

### 設計

1. ツールバーから `imageDir` ボタンを削除（Full/Simple共通）
2. サイドバーフッターに常時、実際の画像保存先パスを表示
3. パスの決定元（File/Settings/Default）を明示
4. サイドバーに設定ボタン（⚙）を配置し、どのモードからでも設定可能に
5. Extension側で解決済みパス（`forceRelativeImagePath` 反映済み）を計算してWebviewに送信

### 実装

#### 新メッセージ `imageDirStatus`

`host-bridge.ts` に新メッセージ型を追加:
```typescript
{ type: 'imageDirStatus'; displayPath: string; source: 'file' | 'settings' | 'default' }
```

#### `editorProvider.ts` - `sendImageDirStatus()` ヘルパー

- `ImageDirectoryManager` に `getFileImageDir(uriKey)` パブリックメソッド追加
- `sendImageDirStatus()` ローカル関数を追加:
  - ソース判定: fileImageDirs/ディレクティブ → `'file'`、VSCode設定 → `'settings'`、未設定 → `'default'`
  - 表示パス計算: `getImageDirectory()` で絶対パス取得後、`forceRelativeImagePath` を反映して相対/絶対パスを決定
- 呼び出しタイミング: 初期化後、設定変更後、外部ドキュメント変更後、setImageDir完了後

#### `webviewContent.ts` - HTML変更

- ツールバーから `<button data-action="imageDir">` を削除
- サイドバーフッターに構造化HTML: `.imagedir-path` + `.imagedir-source` + `.imagedir-settings-btn`（⚙アイコン）

#### `editor.js` - メッセージ受信・表示

- `imageDirDisplayPath` / `imageDirSource` グローバル変数追加
- `imageDirStatus` メッセージハンドラ追加
- `updateStatus()` を更新: パスとソースラベルを常時表示
- ⚙ボタンの `click` イベントで `host.requestSetImageDir()` を呼び出し

#### `styles.css` - レイアウト

- flexレイアウトでパス・ソースラベル・設定ボタンを1行表示
- パスは `text-overflow: ellipsis` で長いパスを省略表示
- `:empty` 非表示ルールを削除（常時表示のため）

#### i18n - 7言語対応

- `imageDirSourceFile` / `imageDirSourceSettings` / `imageDirSourceDefault` を全7言語に追加
