# v0.195.468 — Outliner スコープ改善 + テーマ・i18n・ページ設定

## 変更サマリー

Outlinerモードに対して13件の要件を一括実装。スコープ機能の基盤修正、パンくずナビゲーション、タグクリック検索、7テーマ対応、7言語i18n、ページディレクトリ設定、ページ表示制約を追加。

---

## Phase 1: スコープ基盤修正

### 課題
- `clearSearch()` がスコープをリセットしていた（検索クリア時にスコープが解除される）
- ESCキーでスコープが解除されていた（意図しないスコープ解除）
- ノード削除後にスコープ対象ノードが存在しない状態になりうる
- スコープ設定/解除のショートカットがない
- 右クリックメニューにスコープ操作がない

### 対策
- `clearSearch()` から `currentScope = { type: 'document' }` と `scopeBadge.textContent = ''` を削除
- ESCハンドラの `else if (currentScope.type !== 'document')` ブロックを削除
- `deleteSelectedNodes()`, `handleBackspaceAtStart()` (3パス), コンテキストメニュー削除の計5箇所に `if (currentScope.type === 'subtree' && !model.getNode(currentScope.rootId)) { setScope({ type: 'document' }); }` を追加
- Cmd+] でフォーカスノードにスコープ設定、Cmd+[ でスコープ解除（`e.stopPropagation()` 付き）
- コンテキストメニューに「Scope」「Clear Scope」項目追加

---

## Phase 2: パンくずナビゲーション

### 課題
- スコープ設定時のUI表示が `scopeBadge`（テキストのみ）で貧弱
- スコープの階層構造が見えない
- スコープ解除のUIがない

### 対策
- `scopeBadge` を廃止し、`breadcrumbEl` (`.outliner-breadcrumb`) に置換
- HTML: 検索バーの上にパンくず `<div>` を配置
- CSS: `display: none` → `is-visible` クラスで表示切替、各アイテムは `max-width: 150px` + `text-overflow: ellipsis`
- JS: `updateBreadcrumb()` 関数を新設。`parentId` チェーンで祖先を辿り、各ノードをクリック可能なスコープ切替ボタンとして表示。右端に TOP ボタン（`flex: 1` スペーサーで右寄せ）
- `setScope()` と `renderTree()` の両方から `updateBreadcrumb()` を呼び出し

---

## Phase 3: タグクリック検索

### 課題
- タグ（#tag, @tag）をクリックしても検索できない

### 対策
- `createNodeElement()` 内の `textEl` に `click` イベントリスナー追加
- `document.activeElement === textEl` で focus中のノードを除外（カーソル移動を優先）
- blur中のノードでのみ `.outliner-tag` をクリックで検索ボックスに自動入力＋検索実行

---

## Phase 4: テーマCSS

### 課題
- Outlinerは `:root` のダークテーマ固定で、mdモードの7テーマに対応していない
- hardcoded `rgba()` 値がテーマ切替で変わらない

### 対策
- 5つの新CSS変数追加: `--outliner-text-focus-bg`, `--outliner-code-bg`, `--outliner-tag-hover-bg`, `--outliner-scrollbar-thumb`, `--outliner-scrollbar-thumb-hover`
- hardcoded rgba値を全てCSS変数に置換
- 7テーマ分の `[data-theme="xxx"]` ブロックを追加（各17変数をオーバーライド）
- mdモードの `styles.css` テーマ色を参考に、各テーマで適切な色を設定

---

## Phase 5: i18n

### 課題
- Outlinerの全文字列がハードコードされており、多言語対応していない

### 対策
- 17キー × 7 locale（en, ja, ko, es, fr, zh-cn, zh-tw）を `webviewMessages` に追加
- `outlinerProvider.ts` で `getWebviewMessages()` を呼び出し、config経由でwebviewに渡す
- `outlinerWebviewContent.ts` で `window.__outlinerMessages` としてHTML内に埋め込み
- `outliner.js` で `var i18n = window.__outlinerMessages || {};` として参照
- 全ハードコード文字列を `i18n.key || 'fallback'` 形式に置換

---

## Phase 6: ページディレクトリ設定

### 課題
- ページディレクトリが `./pages` に固定されている

### 対策
- `package.json` に `fractal.outlinerPageDir` 設定追加（デフォルト: `./pages`）
- `getPagesDirPath()` を拡張: (1) mmd JSONの `pageDir` フィールド優先、(2) VSCode設定にフォールバック
- 相対パスはmmdファイルからの相対で解決、絶対パスはそのまま使用
- `setPageDir` メッセージハンドラ追加（`vscode.window.showInputBox` でパス入力 → mmd JSONに保存）

---

## Phase 7: ページ表示制約

### 課題
- outlinerから開いたページで画像ディレクトリが変更可能
- outlinerから開いたページでaddPage機能が表示される

### 対策
- `OutlinerProvider.outlinerPagePaths` static Mapでページファイルパスを追跡
- `handleOpenPage()` でMap登録、`editorProvider.ts` の `onDidDispose` でクリーンアップ
- `editorProvider.ts`: outlinerページ判定時に `imageDirectoryManager.setFileImageDir()` で `{pageDir}/images/` を強制設定
- `setImageDir` メッセージハンドラでoutlinerページの場合は `break` で拒否
- `webviewContent.ts`: `isOutlinerPage` をEditorConfigに追加、`__IS_OUTLINER_PAGE__` でeditor.jsに渡す
- `editor.js`: `IS_OUTLINER_PAGE` が true の場合、`COMMAND_PALETTE_ITEMS` から `addPage` をフィルタ、`imageDirSettingsBtn` を非表示
