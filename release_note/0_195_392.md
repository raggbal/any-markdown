# v0.195.392 - ツールバー3分割レイアウト（固定左右 + スクロール中央）

## 課題

fullモードのツールバーが画面幅に収まらない場合、`<` `>` スクロールボタンで横スクロールできるが、undo/redo（左側）や openInTextEditor/source（右側）のユーティリティボタンまで一緒にスクロールしてしまい、常用ボタンにアクセスしづらい。

## 原因

ツールバー全体が単一の `toolbar-inner` コンテナ内にあり、全ボタングループが同一のスクロール領域に含まれていた。

## 対策

### 設計

ツールバーを3分割レイアウトに変更:

```
.toolbar (flex)
├── .toolbar-fixed--left   固定: openOutline, undo, redo
├── < スクロールボタン
├── .toolbar-inner         スクロール: inline, block, insert
├── > スクロールボタン
└── .toolbar-fixed--right  固定: openInTextEditor, source
```

simpleモードと同じ左右配置を維持しつつ、fullモードでは中央のmarkdown要素系ボタンのみがスクロールする。

### 実装

#### HTML構造変更 (webviewContent.ts, html-generator.ts)
- openOutline + history グループを `toolbar-fixed--left` に移動
- utility グループを `toolbar-fixed--right` に移動
- inline, block, insert グループのみ `toolbar-inner` に残す
- スクロールボタンは `toolbar-inner` の左右に隣接配置

#### CSS変更 (styles.css)
- `.toolbar-fixed` クラス追加（`flex-shrink: 0` で固定幅）
- `.toolbar-scroll-btn` を `position: absolute` から通常フローに変更（`flex-shrink: 0`）
- `.toolbar-group[data-group="utility"]` の `margin-left: auto` 削除（不要に）
- simpleモード: `.toolbar-inner` を `display: none`、`.toolbar-fixed--right` に `margin-left: auto`

#### JS変更なし
- `toolbarInner` の `getElementById` 参照はそのまま有効
- スクロールボタンのオーバーフロー検出ロジックも変更不要
