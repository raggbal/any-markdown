# v0.195.605 リリースノート

## 1. Cross-Outliner コピー/ペースト（ページ・画像対応）

### 課題
同一outliner内でのノードcopy/cut→pasteはページ(.md)や画像も正しく複製/移動されるが、同じNotes内の別outlinerや、別Notesのoutlinerに対してペーストすると、ノードテキストは貼り付けられるものの、ページファイルと画像ファイルのコピー/移動に失敗していた。

### 原因
3つのレイヤーの問題:
1. **`internalClipboard` がJSメモリ内のみ** — 別webviewインスタンスからは参照不可能
2. **`copyPageFile` メッセージハンドラ**が現在のファイルの `pagesDirPath` をソースとして使用 — ファイル切替後はソースファイルのpageDirを参照できない
3. **画像にはファイルコピー機構がない** — パス文字列のコピーのみで実ファイルのコピーもパス変換もなし

### 設計
2層のクリップボード + ホスト側ファイルコピーで解決:
- **Layer 1**: `internalClipboard` に `sourcePageDir` を追加（同一webview内のcross-fileペースト用）
- **Layer 2**: `text/html` クリップボードの `<ul>` タグに `data-outliner-clipboard` 属性でノードメタデータを埋め込み（cross-webviewペースト用）
- **Host-side**: `OutlinerClipboardStore` 静的シングルトンがソースの絶対パスを保持。`copyPageFileCross` / `movePageFileCross` / `copyImagesCross` メッセ��ジでファイルコピー/移動を実行

### 実装

**新規ファイル:**
- `src/shared/outliner-clipboard-store.ts` — 全provider間共有の静的シングルトン

**変更ファイル:**
- `src/webview/outliner.js` — copy/cut: `sourcePageDir`保存 + HTML metadata埋め込み + `saveOutlinerClipboard`送信。paste: HTML fallback + cross-file分岐。単一ノードcopyも対応
- `src/shared/outliner-host-bridge.js` / `notes-host-bridge.js` — 4メソッド追加
- `src/shared/notes-message-handler.ts` — 5つの新caseハンドラ
- `src/outlinerProvider.ts` — 5つの新caseハンドラ

**テスト:** 8件追加 (`outliner-cross-paste.spec.ts`)

## 2. 空ノード+子ありのBackspaceバグ修正

### 課題
子ノードを持つ空ノードでBackspaceを押すと、空ノードだけでなく子ノードも一緒に削除されてしまう。本来、親が消えたら子は同じ位置でデインデント（親のレベルに昇格）すべき。

### 原因
`model.removeNode()` が再帰的に全子孫を削除する仕様。空ノード+子ありの分岐が存在せず、空ノード+子なしの `removeNode()` コードパスに入っていた。

### 対策
`handleBackspaceAtStart()` に空ノード+子あり専用の分岐を追加:
1. 子ノードIDをコピーし `node.children = []` でクリア（再帰削除防止）
2. 親の `children`/`rootIds` から空ノードを除去 + `delete model.nodes[node.id]`
3. 子ノードの `parentId` を空ノードの親に更新し、元の位置（空ノードがあった場所）に挿入

**テスト:** 3件追加 (`outliner-backspace-children.spec.ts`)
