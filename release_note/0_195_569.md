# v0.195.569 リリースノート

## 修正1: コピー/カット時の画像消失

### 課題
ノードを Cmd+C/Cmd+X でコピー/カットし、別ノードに Cmd+V でペーストすると、ノードに添付されていた画像が消失する。

### 原因
`getSelectedNodesData()` が内部クリップボードにテキスト・ページメタデータのみ保存し、`images` 配列を含めていなかった。`pasteNodesFromText()` でのノード復元時に画像データがないため空になっていた。

### 対策
- `getSelectedNodesData()` に `images` フィールドを追加（`nd.images.slice()` でコピー）
- `pasteNodesFromText()` で `clipNode.images` がある場合に `newNode.images` へ復元
- ページメタデータと同パターン: カット→移動（同パス参照）、コピー→複製（同パス参照）

## 修正2: 画像D&Dの操作性改善

### 課題
画像サムネイルのドラッグ&ドロップによる並べ替えが困難。各画像要素ごとのdragover判定のため、画像間の隙間や端ではドロップが反応しない。

### 原因
個別画像の `dragover`/`drop` イベントハンドラで中央判定していたため、マウスが画像要素外（隙間部分）にある時にドロップターゲットとして認識されない。

### 対策
- 個別画像ごとの `dragover`/`drop` を廃止し、コンテナ（`.outliner-images`）レベルのイベントに統一
- `getImageDropIndex(container, clientX, clientY)`: 全画像の左端/右端とマウス位置の距離を計算し、最も近いドロップインデックスを返す
- `showImageDropIndicator()` / `clearImageDropIndicators()` でインジケーター表示を分離
- 画像間の隙間やコンテナ端でもドロップ可能に

### テスト結果
- 全915テスト通過、4スキップ、0失敗 — デグレなし
