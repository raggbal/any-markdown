# v0.195.411 - サイドパネル修正 + Electronウェルカム画面

## 課題

### 課題1: サイドパネルで画像操作が不可（VSCode/Electron共通）
サイドパネル（iframe）内のエディタで画像のペースト（Cmd+V）、ドラッグ&ドロップ、ツールバー画像挿入ができなかった。

### 課題2: サイドパネル「Open in new tab」ボタンが効かない（VSCode/Electron共通）
サイドパネルヘッダーの「Open in new tab」ボタンをクリックしても反応がないように見えた。

### 課題3: Electronパッケージ版でサイドパネルが開かない
DMGからインストールしたElectronアプリで.mdリンクをクリックしてもサイドパネルが表示されなかった。

### 課題4: Electronでファイル未選択時にAction Panelが誤動作
Electronアプリをファイル指定なしで起動すると、`getDocumentDir()`が`process.cwd()`（macOSでは`/`）を返し、ファイル検索やページ作成がルートディレクトリ基準で動作していた。

## 原因

### 課題1の原因
editor.jsのサイドパネルiframeメッセージ中継ハンドラ（`window.addEventListener('message')`）に、画像関連メッセージ3種（`saveImageAndInsert`, `readAndInsertImage`, `insertImage`）の中継処理が未実装だった。

さらに、ホストからの画像保存レスポンス（`insertImageHtml`）が常にメインエディタに適用され、サイドパネルiframeに中継される仕組みがなかった。

### 課題2の原因
ボタンクリック自体は機能していたが、`host.openLinkInTab()`実行後にサイドパネルが閉じないため、新しいタブ/ウィンドウがサイドパネルの背後に開き、ユーザーには何も起きていないように見えた。

### 課題3の原因
`electron/package.json`の`extraResources`設定で、`out/shared`フィルタが`editor-body-html.js`のみに限定されており、`side-panel-host-bridge.js`がDMGパッケージに含まれていなかった。`generateSidePanelHtml()`が`fs.readFileSync`でファイルを読めずにエラーで失敗していた。

### 課題4の原因
VSCode拡張ではCustom Editorが常にドキュメントを持つため問題にならないが、Electronではファイル指定なしでウィンドウを開くことが可能で、`FileManager.getDocumentDir()`が`process.cwd()`にフォールバックしていた。

## 対策

### 設計

**課題1**: `sidePanelImagePending`フラグを導入。サイドパネルからの画像リクエスト時にフラグを立て、ホストからの`insertImageHtml`レスポンス時にフラグを判定してiframeに中継する。

**課題2**: 「Open in new tab」ボタンのクリックハンドラで`host.openLinkInTab()`実行後に`closeSidePanelImmediate()`を呼ぶ。

**課題3**: `extraResources`フィルタに`side-panel-host-bridge.js`を追加。

**課題4**: ファイル未指定時にウェルカム画面を表示し、ユーザーにファイルの選択または作成を強制する。

### 実装

#### 変更ファイル

**src/webview/editor.js**（VSCode/Electron共通）
- サイドパネルiframeメッセージ中継に画像関連3種追加: `saveImageAndInsert`, `readAndInsertImage`, `insertImage`
- `sidePanelImagePending`フラグ追加
- `insertImageHtml`レスポンスハンドラにフラグ判定追加（iframeまたはメインエディタに振り分け）
- 「Open in new tab」ボタンクリック後に`closeSidePanelImmediate()`追加

**electron/package.json**
- `extraResources`の`out/shared`フィルタに`side-panel-host-bridge.js`追加

**electron/src/html-generator.ts**
- `generateWelcomeHtml(theme)`関数追加: テーマ対応のウェルカム画面HTML生成（Open File / Create New File / Recent Files）

**electron/src/preload.ts**
- `window.welcomeBridge`追加（`openFile`, `createFile`, `openRecent`, `getRecentFiles`）

**electron/src/main.ts**
- `createWindow()`でファイル未指定時にウェルカム画面をロード
- 4つのIPC ハンドラ追加: `welcome-open-file`, `welcome-create-file`, `welcome-open-recent`, `welcome-get-recent-files`

**electron/src/settings-manager.ts**
- `getRecentFiles()`メソッド追加
