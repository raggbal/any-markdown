# v0.195.570 リリースノート

## バグ修正: 画像付きノードの単一行コピー/カットでインライン貼り付けになる

### 課題
画像が添付されたノードを1行だけ選択して Cmd+C/Cmd+X → Cmd+V すると、テキストがインライン挿入（カーソル位置に文字列挿入）され、画像が消失する。ページ（MD）が添付されたノードではノードレベルの貼り付けになるのに、画像では動作が異なっていた。

### 原因
`handleNodePaste()` の単一行ペースト分岐条件:
```javascript
if (!clipText.includes('\n') && !hasPageInClipboard) {
```
`hasPageInClipboard` はページメタデータの有無のみチェックし、画像の有無を見ていなかった。画像付きでもページなしの単一行ノードはインライン貼り付けパスに入り、`execCommand('insertText')` でテキストのみ挿入されていた。

### 対策
- `hasPageInClipboard` を `hasMetadataInClipboard` にリネーム・拡張
- 判定条件を `cn.isPage || (cn.images && cn.images.length > 0)` に変更
- 画像がある場合はインライン貼り付けをスキップし、ノードレベルの貼り付け（`pasteNodesFromText()`）にフォールスルー
- ページメタデータと画像メタデータを統一的に「メタデータ付きノード」として扱う

### テスト結果
- Outliner全176テスト通過 — デグレなし
