# v0.195.409 - Action Panel バグ修正 (hover/keyboard競合、i18n、click、IME、h1)

## 課題

v0.195.407で追加したAction Panelに以下のバグがあった:

1. オートコンプリート候補で↓キーとマウスホバーを併用すると、2つのアクティブ行が表示される
2. メニュー項目の日本語ローカライズ不足（Add Page / Add Page at Path等）
3. メニュー・パス入力・リンク名のすべてでclickが動作しない（Enterのみ）
4. リンク名入力でIME変換中のEnterで確定してしまう
5. 新規ファイルのh1がファイル名になっている（リンク名であるべき）
6. addPageがコマンドパレットの下の方にある

## 対策

### hover/keyboard競合（CSS + JS）

- CSS: `.action-panel-item:hover` と `.action-panel-result-item:hover` を削除
- JS: メニュー項目とオートコンプリート候補に `mousemove` ハンドラを追加。マウス移動時に `selectedIndex` を更新して `.selected` クラスを付け替え
- 結果: キーボードとマウスが常に1つの `selectedIndex` を共有

### i18n

- `addPage` → 「ページ挿入」、`addPageAuto` → 「ページ追加」、`addPageAtPath` → 「パス指定でページ追加」等を en.ts / ja.ts に追加

### click動作

- メニュー項目・OKボタンの `click` イベントに `mousedown` で `preventDefault` を追加し、inputからのフォーカス移動を防止
- パス入力のOKボタン・リンク名のOKボタンすべてで動作確認

### IME対応

- リンク名入力の `compositionstart`/`compositionend` でフラグ管理、`keydown` でフラグチェック済み（既存実装で対応済みだが動作確認）

### h1反映

- `createPageAtPath` の初期内容を `# {basename}` から `# {linkName}` に変更（リンク名がパスと同じ場合はh1なし、既存ファイル選択時はh1変更なし）
- `finalizeAddPage` から `updatePageH1` メッセージで新規作成ファイルのh1をリンク名に更新

### addPage位置

- コマンドパレットの `PALETTE_ITEMS` 配列でaddPageを先頭に移動
