# v0.195.550 リリースノート

## 全エディタ 複数パネル間同期

### 課題

同じファイルをVSCodeの複数タブグループで開いて編集した場合、片方の変更がもう一方に反映されなかった。

| エディタ | 原因 |
|---------|------|
| Markdown (.md) | `supportsMultipleEditorsPerDocument: false` でVSCodeが2つ目のパネル作成を制限 |
| Outliner standalone (.out) | 同上 |
| Notes (.note) | `this.panel`（単一参照）で管理。2つ目のパネルが開くと1つ目の管理が外れる（孤児化） |

### 対策

#### Part 1: Markdown / Outliner standalone (extension.ts — 2行変更)

`supportsMultipleEditorsPerDocument` を `false` → `true` に変更。

`resolveCustomTextEditor` 内の全状態変数（`isApplyingOwnEdit`, ハンドラ等）はメソッド内の**ローカルクロージャ**で管理されているため、VSCodeが同じドキュメントで複数回呼び出すと、各パネルが完全に独立した状態を持つ。

同期メカニズム:
```
Panel A 編集 → isApplyingOwnEdit_A = true → WorkspaceEdit
→ onDidChangeTextDocument 発火
→ Panel A: isApplyingOwnEdit_A = true → skip（自分の編集）
→ Panel B: isApplyingOwnEdit_B = false → 検知 → webviewに通知 ✓
```

追加コード不要。既存のクロージャ構造がそのまま複数パネル同期として機能する。

#### Part 2: Notes editor (notesEditorProvider.ts — 全面リファクタリング)

**アーキテクチャ変更**: `this.panel`/`this.fileManager`等のインスタンスプロパティを全廃止し、Markdown/Outlinerと同じ**ローカル変数+クロージャ方式**に統一。

Before:
```typescript
class NotesEditorProvider {
    private panel: WebviewPanel | undefined;       // 単一参照
    private fileManager: NotesFileManager | undefined;  // 単一参照
    private currentFolderPath: string | undefined;
    private disposables: Disposable[] = [];
    private folderWatcher: FileSystemWatcher | undefined;
}
```

After:
```typescript
class NotesEditorProvider {
    constructor(private context: ExtensionContext) {}
    // 全てopenNotesFolder内のローカル変数
    // panel, fileManager, folderPath, sidePanel, disposables, folderWatcher
}
```

各パネルが独自の `fileManager` を持つため、`isWriting` フラグがパネル独立。Panel Aが書き込むとPanel Aの`fileManager.isWriting = true`だが、Panel Bの`fileManager.isWriting = false`のままなので、FileSystemWatcher経由で変更を正しく検知する。
