# v0.195.449 - Outlinerタグ脱出時のスペース挿入修正

## 課題

Outliner（.mmd）モードで、タグ（`#tag` / `@tag`）内にカーソルがある状態でSpaceキーを押すと：
- 1回目のSpace: タグ領域からカーソルは脱出するが、スペース文字が表示されない
- 2回目のSpace: スペース文字が入力される

1回のSpace押下で「タグ脱出 + スペース挿入」を同時に行いたい。

## 原因

`contenteditable` 要素の末尾にある通常スペース（U+0020）はブラウザが描画しない。

タグ `#tag` がノードの末尾にある場合、脱出コードが `document.createTextNode(' ')` で挿入したスペースは contenteditable 要素の末尾空白となり、ブラウザのCSS仕様に従い視覚的に表示されなかった。

DevTools開放時はブラウザの描画タイミングが変わるため動作していたが、通常時は末尾空白として非表示になっていた。

## 対策

### 設計

末尾空白が描画されない問題を回避するため、NBSP（`\u00A0` / Non-Breaking Space）を使用する方式に変更。NBSPはブラウザが必ず描画する。モデル（データ）側には通常スペースとして保存し、表示時のみNBSPを使う。

### 実装（`src/webview/outliner.js`）

1. **Spaceキーハンドラ（case ' '）**: タグ脱出時のスペースノードを `\u00A0` で生成するよう変更
2. **`getPlainText()`**: NBSPを通常スペースに正規化（`\u00A0` → `' '`）してモデルに保存
3. **`renderInlineText()`**: innerHTML再構築時、末尾スペースをNBSPに変換。これにより `input` イベントによるDOM全再構築後もスペースが消えない
