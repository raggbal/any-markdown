# v0.195.603 リリースノート

## 新機能: アプリ内リンク機能

Notes editor内のOutlinerノードやMarkdownページに対して、一発でジャンプ可能なアプリ内リンクを生成・貼り付け・クリック遷移する機能を追加。

---

## 1. ノードリンク

### 概要
Outlinerノードを右クリック「アプリ内リンクをコピー」でMarkdownリンク形式のリンクを生成。貼り付けてクリックすると対象noteを開き、outlinerファイルを切り替え、ノードにジャンプする。

### URI形式
```
fractal://note/{noteFolderName}/{outFileId}/{nodeId}
```

### 実装
- **`src/webview/outliner.js`**: コンテキストメニューに「アプリ内リンクをコピー」追加（Notes modeのみ、単一選択のみ）
- 表示テキスト: `stripInlineMarkers()` + `#tag/@tag` 除去
- **`src/extension.ts`**: `parseFractalLink()` URI解析、`fractal.navigateInAppLink` コマンド登録
- **`src/notesEditorProvider.ts`**: `navigateToLink()` → webviewに `notesNavigateInAppLink` メッセージ送信
- **`src/shared/notes-message-handler.ts`**: `notesNavigateInAppLink` ハンドラ（ファイル切替 + `jumpToNodeId`）
- **`src/webview/outliner.js`**: `notesNavigateInAppLink` 受信 → sidepanel即座閉じ + `notesHostBridge.jumpToNode()`
- **fractal://検出（5箇所）**: `editorProvider.ts`, `outlinerProvider.ts`, `notes-message-handler.ts`, `sidePanelManager.ts` の各openLinkハンドラ

### 課題と対策
- **jumpToNodeId遅延**: renderTree()完了前にジャンプすると失敗。100ms→300msに増加
- **sidepanel競合**: closeSidePanel()のsetTimeout(200ms)がjumpToNodeと競合。closeSidePanelImmediate()に変更

## 2. ページリンク

### 概要
sidepanelヘッダーの🔗ボタンでページへのアプリ内リンクを生成。クリックすると現在のnote/outlinerを維持したままsidepanelで指定mdファイルを表示。

### URI形式
```
fractal://note/{noteFolderName}/{outFileId}/page/{pageId}
```

`page/` プレフィックスでノードリンクと区別。nodeIdは不要（cmd+x→cmd+vでnodeId変更、pageIdは不変のため）。

### 実装
- **`src/shared/editor-body-html.js`**: sidepanelヘッダーにリンクアイコンボタン追加（`display:none`、Notes modeで表示）
- **`src/webview/outliner.js`**: ボタンクリック → H1テキスト優先（なければノードtext）でリンク生成
- **`src/notesEditorProvider.ts`**: `resolvePagePath()` でディスクから直接パス解決（noteが開いていなくても可）、`openPageInCurrentPanel()` で現在visibleなパネルのsidepanelで表示
- **`openPanels` Map拡張**: `{ panel, postMessage, fileManager, openPage }` — `openPage` はSidePanelManager.openFile()へのクロージャ参照

### 設計判断
- note移動・outliner切替は行わない（ユーザー要件: md編集で行がずれるため行指定は非対応）
- 別noteフォルダのmdも現在のnoteのsidepanelで開ける

## 3. Markdown右クリックメニュー

### 概要
全Markdown editor（単体.md + sidepanel）に右クリックコンテキストメニュー（Cut/Copy/Paste）を追加。

### 課題と対策

#### VSCode webviewでcontextmenuイベントが要素レベルに届かない
- **原因**: VSCode webview固有の制約。`editor.addEventListener('contextmenu', ...)` ではイベント到達しない
- **対策**: `document.addEventListener('contextmenu', ...)` をdocument-levelで登録。`editor.contains(e.target)` ガードでインスタンス分離を維持（クロージャ設計保持）

#### VSCode webviewでCSSクラスが適用されない
- **原因**: `styles.css` の `.editor-context-menu` クラスが適用されず、`z-index: auto`、位置もドキュメントフロー上
- **対策**: 全スタイルをインラインCSS (`style.cssText`) で設定。テーマCSS変数は `getComputedStyle(document.documentElement)` で動的取得

#### メニュークリックでSelectionが消失 (M-11)
- **原因**: メニュー項目クリック時にeditorからフォーカスが移動しSelectionが失われる
- **対策**: メニュー表示時に `sel.getRangeAt(0).cloneRange()` でRange保存、操作前に `editor.focus({preventScroll: true})` + `sel.addRange(savedRange)` で復元

#### Paste処理
- **対策**: `document.execCommand('paste')` でネイティブpasteイベントハンドラを発火（Cmd+Vと同じコードパス）

## 4. 基盤整備

### i18n (7言語)
`copyInAppLink`, `contextCut`, `contextCopy`, `contextPaste` を全7言語に追加。

### noteFolderName注入
- `notesWebviewContent.ts`: `<div class="notes-layout" data-note-folder-name="${config.folderName}">`
- `notesEditorProvider.ts`: `folderName: path.basename(folderPath)` を config に追加

### currentOutFileId追跡
- `notes-file-panel.js`: `getCurrentOutFileId()` — fileListから現在ファイルのIDを返す

### NotesFolderProvider拡張
- `getFolders()`: 登録済みフォルダパス一覧を返す（リンク先フォルダ名の解決に使用）
