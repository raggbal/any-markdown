# v0.195.456 - ペースト位置がずれるバグ修正

## 課題

空ノードでCmd+Vすると、貼り付けたノードが本来の位置ではなく兄弟リストの末尾に追加される。カーソル位置の1つ上の行が異なるインデントの場合に必ず発生。

### 再現例

```
- a
  - b
- |  ← ここでペースト
- c
```

結果: ペーストしたノードが c の後に追加される。

## 原因

空ノードの挿入位置計算で `getPreviousVisibleId()` を使用していた。この関数はDFS順の前ノードを返すため、異なる深さの子ノード（例: `b`）が返る。`prevNode.parentId === parentId` の兄弟判定が失敗し、`insertAfter = null` → 末尾追加になっていた。

## 対策

DFS前ノードではなく、**親の `children` 配列（ルートなら `rootIds`）** から直接インデックスで前の兄弟を取得するよう変更。

```javascript
var siblings = parentId ? (model.getNode(parentId).children || []) : model.rootIds;
var sibIdx = siblings.indexOf(nodeId);
var insertAfterForEmpty = sibIdx > 0 ? siblings[sibIdx - 1] : null;
```
