# v0.195.401 - サイドパネルの外部変更リアルタイム反映

## 課題

サイドパネル（iframe方式）で開いたMarkdownファイルが、外部エディタ（Claude、別のVSCodeウィンドウ等）から変更された場合、変更がリアルタイムに反映されなかった。メインエディタでは正常に外部変更が検知・反映されるが、サイドパネルには同等の仕組みがなかった。

## 原因

メインエディタには `FileSystemWatcher` による外部変更検知→webviewへの `update` メッセージ送信の仕組みがあるが、サイドパネルで開いたファイルには `FileSystemWatcher` が設定されておらず、外部変更を検知する仕組みが一切存在しなかった。

### メインエディタの既存フロー
```
ファイル変更 → FileSystemWatcher.onDidChange → readFile → postMessage({type:'update'}) → webview更新
```

### サイドパネルの問題フロー
```
ファイル変更 → (何も起きない) → iframe は古い内容のまま
```

## 対策

### 設計

サイドパネル専用の `FileSystemWatcher` を追加し、メインエディタと同様に外部変更を検知→iframe内のeditor.jsに `update` メッセージを中継する仕組みを構築。

自己書き込みによる無限ループ防止のため、`isSidePanelSaving` フラグによるガードを実装。

### 実装

#### 変更ファイル

1. **src/shared/host-bridge.ts** — `notifySidePanelClosed()` メソッドをインターフェースに追加
2. **src/shared/vscode-host-bridge.js** — 同メソッドの VSCode 実装追加
3. **src/shared/test-host-bridge.js** — 同メソッドのテスト用実装追加
4. **src/editorProvider.ts**:
   - `setupSidePanelFileWatcher(filePath)` — サイドパネルファイル用 FileSystemWatcher 作成
   - `disposeSidePanelFileWatcher()` — watcher 破棄
   - `isSidePanelSaving` フラグ — `saveSidePanelFile` 時の自己書き込みガード（300msタイマーでリセット）
   - `openSidePanel` 送信時（2箇所: openLink, sidePanelOpenLink）で watcher 作成
   - `sidePanelClosed` メッセージハンドラで watcher 破棄
   - `onDidDispose` で watcher 破棄
5. **src/webview/editor.js** — `closeSidePanelImmediate()` で `host.notifySidePanelClosed()` 呼び出し

#### メッセージフロー（修正後）

```
外部変更 → FileSystemWatcher.onDidChange
  → readFile
  → webviewPanel.postMessage({type:'sidePanelMessage', data:{type:'update', content}})
  → editor.js が iframe に中継 (contentWindow.postMessage)
  → iframe 内の editor.js が 'update' メッセージを処理 → DOM更新
```
