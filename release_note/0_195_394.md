# v0.195.394 - Markdownリンクの開き方設定（タブ / サイドパネル）

## 課題

Markdownファイル内の内部リンク（`[text](./other.md)`）をクリックした際、リンク先のMarkdownファイルがVSCodeのデフォルトテキストエディタで開かれていた。Notionのサイドピークのように、右パネルでリンク先を開いてWYSIWYG編集したいという要望。

また、内部リンクのパス解決がワークスペースルート基準になっており、ドキュメントからの相対パスが正しく解決されないバグがあった。

## 原因

1. `editorProvider.ts` の `openLink` ハンドラが `.md` ファイルを判別せず、全ての内部リンクを `vscode.open`（デフォルトエディタ）で開いていた
2. パス解決が `vscode.workspace.getWorkspaceFolder(document.uri)` 基準で、ドキュメントディレクトリ基準ではなかった

## 対策

### 設計

- `fractal.linkOpenMode` 設定を追加（`"tab"` / `"sidePanel"`、デフォルト: `"sidePanel"`）
- `.md` / `.markdown` ファイルリンクはAny Markdownエディタで開く
- `"sidePanel"` → `ViewColumn.Beside`（右パネル）、`"tab"` → `ViewColumn.Active`（現在のタブ）
- パス解決をドキュメントディレクトリ基準に修正

### 実装

1. **package.json**: `fractal.linkOpenMode` 設定を `configuration.properties` に追加
2. **editorProvider.ts**: `openLink` ハンドラを修正
   - `vscode.Uri.joinPath(document.uri, '..')` でドキュメントディレクトリ基準のパス解決
   - `.md` / `.markdown` 拡張子判定 → `vscode.openWith` + `fractal.editor` で開く
   - `linkOpenMode` 設定に応じて `ViewColumn.Beside` / `ViewColumn.Active` を切替
   - `.md` 以外のファイルは従来通り `vscode.open` で開く
