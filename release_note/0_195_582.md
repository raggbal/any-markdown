# v0.195.582 リリースノート

## 修正1: Outliner — URL内の@をタグ化しない

### 課題
Outliner editorで `https://example.com/@user` のようなURL内の `@` が自動的にタグとして検出・ハイライトされていた。URL内の `@` はタグではなくURLの一部であり、タグとして扱うべきではない。

### 原因
タグ検出の正規表現 `(?<![&#\w\p{L}])([#@][\w\p{L}][\w\p{L}-]*)` がURL内の `@` にもマッチしていた。lookbehind `(?<![&#\w\p{L}])` は `/` の直後の `@` を除外できない（`/` は `\w` でも `\p{L}` でもないため）。

### 対策

**設計**: タグ正規表現を適用する前に、URLテキストをプレースホルダーに一時退避し、タグ変換後に復元する方式を採用。3箇所すべてに適用。

**実装** (3箇所):

1. **`parseTags()`** (`outliner-model.js`): `text.replace(/https?:\/\/\S+/g, '')` でURLを除去してからタグ抽出。parseTagsはタグ名のみを返すのでURL除去で十分。

2. **`renderInlineText()`** (`outliner.js`): Markdownリンク `[text](url)` → `<a>` 変換と生URL `https://...` の両方をプレースホルダー `\x00LINK{n}\x00` に退避。タグ変換後にプレースホルダーを元に戻す。

3. **`renderEditingText()`** (`outliner.js`): 生URLをプレースホルダーに退避してからタグ変換。編集モードではMarkdownリンクは生テキスト表示のため、URLパターンのみ退避。

## 修正2: サイドパネル — Cmd+F検索ウィンドウのCSS修正

### 課題
Outliner/NotesからサイドパネルでMarkdownエディタを開いた際、Cmd+Fで表示される検索ウィンドウが正しい位置に表示されない。standalone版では正常に動作する。

### 原因
`.search-replace-box` が `position: fixed` でビューポート基準で配置されていた。standalone版ではビューポート全体がエディタなので問題ないが、サイドパネルは `.side-panel` (`position: absolute`) 内の `.side-panel-editor-root` 内にあるため、`position: fixed` だとビューポートの右上（サイドパネル外）に表示されてしまう。

### 対策

**設計**: サイドパネル内の検索ボックスのみ `position: absolute` に切替え、親要素 `.editor-wrapper` (`position: relative`) を基準に配置する。

**実装**: `styles.css` にオーバーライドルールを追加:
```css
.side-panel-editor-root .search-replace-box {
    position: absolute;
    top: 10px;
    right: 10px;
}
```
