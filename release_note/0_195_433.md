# v0.195.433 - リスト内画像の空判定修正 + 画像挿入後のBR除去 + バレット消失修正

## 課題

### 課題1: リスト項目内の画像でEnter押下時にoutdentされる
- リスト項目に画像のみがある状態でEnterを押すと、新しい兄弟liが作成されず、空項目として扱われoutdent（段落変換）されてしまう
- 例: `- ![img](path)` でEnter → リストが壊れて `- ` のみになる

### 課題2: リスト項目への画像貼り付け後に視覚的ずれが発生
- 画像をリスト項目に貼り付けると、バレットからずれた位置に表示される
- mdファイルを開き直すと正しく表示される

## 原因

### 課題1の原因
`isEmptyItem`判定ロジックが4箇所（Enter×1、Backspace×3）に存在し、いずれも`<li>`の子ノードを走査して`directTextContent`を算出している。`<img>`要素の`textContent`は空文字列（`""`）であるため、画像のみのリスト項目が「空」と誤判定されていた。

```javascript
// 修正前: imgのtextContentは""なので空扱い
if (tag !== 'ul' && tag !== 'ol' && tag !== 'input' && tag !== 'br') {
    directTextContent += child.textContent; // img → ""
}
const isEmptyItem = directTextContent === ''; // true（誤判定）
```

### 課題2の原因
`insertImageHtml`ハンドラで`range.insertNode(img)`実行後、contenteditableがカーソル位置確保のため自動的に`<br>`を追加する。この`<br>`がリスト項目内に残り、画像の後に余分な改行が表示されて視覚的にずれる。mdファイル再読み込み時はmarkdown→HTML変換で`<br>`が生成されないため正常表示される。

## 対策

### 設計
1. `isEmptyItem`判定に`hasContentElement`フラグを追加し、`<img>`要素を「コンテンツあり」として認識する
2. 画像挿入後、リスト項目内の不要な末尾`<br>`を除去する

### 実装

#### isEmptyItem修正（4箇所）
- `editor.js` 行6187付近（Backspaceハンドラ）
- `editor.js` 行6914付近（Enterハンドラ）
- `editor.js` 行9085付近（Backspaceハンドラ）
- `editor.js` 行10152付近（Backspaceハンドラ）

```javascript
let hasContentElement = false;
// ...
if (childTag === 'img') {
    hasContentElement = true;
} else if (childTag !== 'ul' && ...) {
    directTextContent += child.textContent;
}
const isEmptyItem = directTextContent === '' && !hasContentElement;
```

#### 画像挿入後のBR除去（1箇所）
- `editor.js` `insertImageHtml`ハンドラ内

```javascript
const parentLi = img.closest('li');
if (parentLi) {
    const nextSib = img.nextSibling;
    if (nextSib && nextSib.nodeName === 'BR') {
        nextSib.remove();
    }
}
```

## 課題3: リスト項目内の画像でバレットが消失する

### 問題

リスト項目に画像がある場合、バレット（`::marker`）が消失する。Tab操作後にバレットが二重表示（`- -`）される。

### 原因

`.editor img`に`display: block`が設定されていた。`<li>`内の`<img>`がblock表示になるとリストマーカーの行から分離し、マーカーは画像の上の空行に表示されて視覚的に消失する。

### 対策

`styles.css`に`.editor li > img { display: inline; margin: 0.5em 0; }`を追加。`<li>`直下の画像のみinline表示にし、バレットと同じ行に表示されるようにした。段落内等の画像は従来通り`display: block`を維持。
