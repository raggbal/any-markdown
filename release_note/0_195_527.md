# v0.195.527: Notes S3 Sync 機能追加

## 概要

Notes機能にAWS S3同期機能を追加。左パネルにS3タブを新設し、3つの同期操作をGUIから実行可能にした。

## 課題

- ノートデータのクラウドバックアップ手段がなかった
- 複数環境間でのノートデータ同期ができなかった

## 設計判断

### AWS CLI vs AWS SDK

AWS SDK（@aws-sdk/client-s3）ではなく、AWS CLI（`aws s3 sync`）をchild_process.spawnで実行する方式を採用。

理由:
- VSIXサイズ影響ゼロ（SDKだと+2.5MB、56%増）
- `aws s3 sync --delete`で差分転送・並列転送・リトライが内蔵
- SDKにはsyncコマンド相当がなく、diffロジックの自前実装が必要
- 認証情報は環境変数でプロセスに渡すためファイルに残らない

トレードオフ: AWS CLIのインストールが前提（パワーユーザー向け機能のため許容）

### 設定の配置

- AWS認証情報（accessKeyId, secretAccessKey, region）→ VSCode settings.json（全フォルダ共通）
- S3バケットパス → outline.note（フォルダごとに異なるパス）

### デグレ防止

- `NoteStructure`への`s3BucketPath`追加はoptional → 既存outline.noteに影響なし
- `NotesPlatformActions`のS3メソッドは全てoptional(`?`) → 既存実装に変更不要
- UIは既存Notes/Searchタブの隣にS3タブを追加 → 既存タブに影響なし

## 実装

### 新規ファイル
- `src/notes-s3-sync.ts` — S3同期エンジン。`child_process.spawn`でAWS CLIを実行し、stdoutを行単位でパースして進捗報告

### 変更ファイル（7ファイル）
- `package.json` — s3AccessKeyId, s3SecretAccessKey, s3Region設定追加
- `src/shared/notes-file-manager.ts` — NoteStructureにs3BucketPath追加 + getter/setter
- `src/shared/notes-message-handler.ts` — S3メッセージ5種 + NotesPlatformActions拡張
- `src/shared/notes-host-bridge.js` — S3ブリッジメソッド7個追加
- `src/shared/notes-body-html.js` — S3タブHTML/CSS追加
- `src/shared/notes-file-panel.js` — S3タブUI制御（ボタン、確認ダイアログ、進捗表示）
- `src/notesEditorProvider.ts` — S3プラットフォームアクション + runS3Operation + getS3Config

### 3つの操作

1. **Sync (Backup)**: `aws s3 sync <local> s3://<bucket> --delete` — 差分のみ転送
2. **Remote Delete & Upload**: `aws s3 rm --recursive` → `aws s3 cp --recursive` — S3全削除後にアップロード
3. **Local Delete & Download**: ローカル全削除(fs) → `aws s3 cp --recursive` — ローカル全削除後にダウンロード

### メッセージフロー

```
Webview(notes-file-panel.js) → notes-host-bridge.js → notesEditorProvider.ts → notes-s3-sync.ts → AWS CLI
                             ← notesS3Progress ←
```

### Local Delete & Download後のリカバリ

ローカルファイルが入れ替わるため、fileManagerをdispose→再作成し、structure再読込→ファイルリスト更新→最初のファイルを開く処理を実装。
