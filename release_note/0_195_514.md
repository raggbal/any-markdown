# v0.195.514 リリースノート

## 概要
Notes機能に5つの新機能を追加。

---

## 1. `.note` → `outline.note` リネーム

### 課題
隠しファイル `.note` はファイルマネージャで非表示になり、ユーザーが認識しにくい。

### 対策
- `getNoteFilePath()` を `outline.note` に変更
- `loadStructure()` に自動マイグレーション追加: 旧 `.note` が存在する場合 `fs.renameSync` で `outline.note` にリネーム
- 影響ファイル: `src/shared/notes-file-manager.ts`

---

## 2. Notes フル検索機能

### 課題
Notes内の複数 .out/.md ファイルを横断的に検索する手段がなかった。

### 設計
- **ストリーミング検索**: `searchFilesStreaming()` でファイル単位で逐次結果送信（notesSearchStart → notesSearchPartial × N → notesSearchEnd）
- **検索対象**: .out ファイルの全ノード text/subtext + フォルダ直下 .md + 各 .out の pages 内 .md
- **検索オプション**: Match Case / Whole Word / Regex
- **ジャンプ**: 検索結果クリック → `notesJumpToNode` → `updateData` + `jumpToNodeId` → outliner.js で親展開+スクロール+ハイライト(2秒フェードアウト)

### 実装
- `notes-file-manager.ts`: `searchFilesStreaming()`, `buildSearchRegex()`, `findMatches()`, `searchMdFile()`
- `notes-message-handler.ts`: `notesSearch`, `notesJumpToNode` ハンドラ
- `notes-host-bridge.js`: `search()`, `jumpToNode()`, `onSearchStart/Partial/End()`
- `notes-body-html.js`: 検索ボタン(虫眼鏡) + 検索パネル(入力/オプション/結果リスト) HTML/CSS
- `notes-file-panel.js`: 検索UI制御(`toggleSearchMode`, `executeSearch`, ストリーミング結果描画, `highlightSearchText`)
- `outliner.js`: `jumpToAndHighlightNode()` — scope解除→親展開→renderTree→スクロール→ハイライト→フォーカス
- `outliner.css`: `.outliner-search-jump-highlight` CSS

---

## 3. Daily Notes 機能

### 課題
日次メモを素早く作成・ナビゲーションする手段がなかった。

### 設計
- **ホスト側ノード操作**: `ensureDailyNode()` で年→月→日の3階層ノードをJSON直接操作で作成。webview側ではなくホスト側で完結させることで、`Outliner.init()` 前にデータを完成させる。
- **Scope In**: `updateData` メッセージに `scopeToNodeId` を追加。outliner.js で `setScope({ type: 'subtree', rootId })` を呼び出し。
- **日付ナビゲーション**: `< >` ボタンで前日/翌日。webview側で `dailyCurrentDate` を追跡し、scope先ノードの年/月/日テキストからカレンダー日付を復元。

### 実装
- `notes-file-manager.ts`: `ensureDailyNotesFile()` (dailynotes.out作成+outline.note登録), `ensureDailyNode()`, `findChildByText()`, `addNodeToData()`
- `notes-message-handler.ts`: `notesOpenDailyNotes`, `notesNavigateDailyNotes` ハンドラ
- `notes-host-bridge.js`: `openDailyNotes()`, `navigateDailyNotes(dayOffset, currentDate)`
- `notes-body-html.js`: Daily Notes行(`< > Today 📅`)
- `notes-file-panel.js`: ボタンクリックハンドラ + `dailyCurrentDate` 追跡
- `outliner.js`: `scopeToNodeId` 対応(setScope呼び出し)

## 4. Notes 左パネル幅D&Dリサイズ (前バージョンで実装済み、本ビルドに含む)

### 課題
左ファイルパネルの幅が固定で調整できなかった。

### 実装
- `NoteStructure` に `panelWidth?: number` 追加
- CSS変数 `--notes-panel-width` + リサイズハンドル
- `notes-file-panel.js`: mousedown/mousemove/mouseup でD&D制御、min 140px / max 50%
- `outline.note` に幅を永続化

---

## 5. サイドパネル幅D&Dリサイズ

### 課題
サイドパネル(ページ表示用)の幅が固定で、Expandのみだった。

### 設計（致命的設計ポイント: syncData上書き問題）
- `.out` への直接書き込みは `syncData` でファイル全体が上書きされ消失する
- **解決策**: `pageDir` / `searchFocusMode` と同じパターンで `sidePanelWidthSetting` を webview変数として保持し、`syncToHostImmediate()` で毎回付与
- host bridge への追加メソッド不要、message handler への case 追加不要

### 実装
- `styles.css`: `.side-panel-resize-handle` CSS (position: absolute, left: -3px, z-index: 101)
- 3箇所にリサイズハンドルHTML追加: `notesWebviewContent.ts`, `outlinerWebviewContent.ts`, `editor-body-html.js`
- `outliner.js`: `setupSidePanelResize()`, `sidePanelWidthSetting` in `syncToHostImmediate()`, Expandボタン連携, `openSidePanel()`で初期幅復元
- `editor.js`: 同等のリサイズコード(セッション内のみ、永続化なし)
