# v0.195.454 - ペースト二重挿入・1行ペースト修正

## 課題1: 二重ペースト

Cmd+Vで貼り付けると、カスタム処理とブラウザデフォルトの両方が動作し、内容が二重に挿入される。

### 原因

`keydown` イベントの `e.preventDefault()` がVSCode webview内で `paste` イベントを完全にブロックしない。`navigator.clipboard.readText()` の非同期処理中にブラウザデフォルトのペーストが発火。

### 対策

`keydown` でのCmd+Vハンドラを廃止し、`paste` イベントリスナーで直接処理する方式に変更。`e.clipboardData.getData('text/plain')` で同期的にクリップボードを取得。

## 課題2: 空ノードにペーストすると先頭行が入らない

空ノードでCmd+Vすると、空ノードが残ったまま、その下から貼り付けが開始される。

### 原因

複数行ペースト時、最初の行を現在ノードの後（`afterId = nodeId`）に挿入していた。

### 対策

空ノードの場合: ノードを削除してから同じ位置に全行をインデント構造付きで挿入。テキストありノードの場合: 現在ノードの後に全行を挿入。
