# v0.195.393 - エディタボディHTML共通化（VSCode/Electron）

## 課題

`src/webviewContent.ts`（VSCode）と `electron/src/html-generator.ts`（Electron）で HTML テンプレート（~150行）が重複管理されていた。VSCode側でサイドバーフッターを整理（f77d897: `sidebar-status-mode` 削除、imagedir UI 追加）したが、Electron側に反映されず、以下の問題が発生:

- Electron版でサイドバーの画像ディレクトリ設定UI（歯車ボタン、パス表示）が表示されない
- 廃止済みの `sidebar-status-mode`（Live Preview）が Electron に残っている
- ツールバーの `imageDir` ボタンが Electron に残っている（サイドバーに移動済み）

## 原因

HTML テンプレートが2ファイルに重複しており、片方の変更がもう片方に手動反映されていなかった。Electron の `tsconfig.json` は `rootDir: "./src"` のため、`../../src/shared/` の TypeScript import が不可能という制約もあった。

## 対策

### 設計

既存の `shared/*.js` パターン（`vscode-host-bridge.js` 等）に倣い、plain JS + CommonJS モジュールとして共通ボディHTML生成関数を抽出。

- `src/shared/editor-body-html.js`: sidebar + toolbar + editor + search の HTML 生成を一元管理
- VSCode: `require('./shared/editor-body-html')` で直接 import
- Electron: `getResourcePath('out/shared/editor-body-html.js')` + `require()` で動的読み込み
- 本番ビルド: `extraResources` に共通モジュールを追加

### 実装

| ファイル | 変更 |
|---------|------|
| `src/shared/editor-body-html.js` | 新規: 共通ボディHTML生成関数 `generateEditorBodyHtml(messages, platform)` |
| `src/webviewContent.ts` | インラインHTML (~100行) を共通関数呼び出しに置換 |
| `electron/src/html-generator.ts` | インラインHTML (~100行) を共通関数呼び出しに置換 |
| `electron/package.json` | `extraResources` に `out/shared/editor-body-html.js` 追加 |

### 共通関数のインターフェース

```javascript
generateEditorBodyHtml(messages, platform)
// messages: Record<string, string> - i18n メッセージ（msg.xxx || '' フォールバック内蔵）
// platform: string - process.platform ('darwin' | 'win32' | 'linux')
// returns: string - <div class="container">...</div> の HTML文字列
```

### copy-webview.js による自動コピー

既存の `scripts/copy-webview.js` が `src/shared/*.js` を `out/shared/` に自動コピーするため、新ファイルも `npm run compile` で自動的に `out/shared/editor-body-html.js` に配置される。追加のビルドスクリプト変更は不要。
