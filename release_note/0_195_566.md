# v0.195.566 リリースノート

## 修正1: Outliner Undo/Redo 根本修正（v0.195.563以前から存在するバグ）

### 課題
Outliner の undo/redo に複数の根本的バグが存在:
1. Notes でファイル切替後、undo/redo ボタンが永久に disabled になる
2. テキスト入力（文字入力）で undo スナップショットが作成されない → 複数ノードの編集が1つの undo ステップにまとまる
3. 初期状態で undo ボタンが active になる（編集前なのに）
4. undo 後に編集すると undo が効かず redo のみ active になる

### 原因
1. `updateData`（ファイル切替）でスタッククリア後に `saveSnapshot()` が呼ばれず、初期スナップショットが存在しない
2. テキスト入力の `input` イベントハンドラが `scheduleSyncToHost()` のみ呼び、`saveSnapshot()` を呼ばない
3. `init()` で `saveSnapshot()` が undoStack に初期状態を push → `updateUndoRedoButtons()` が undo を enabled にする
4. テキスト入力がスナップショットを作らないため、undo 後の編集が undoStack に反映されない

### 対策

**baselineSnapshot 方式の導入:**

| 概念 | 役割 |
|------|------|
| `baselineSnapshot` | 「これ以上戻れない」初期状態。undoStack には入れない → ボタン disabled |
| `saveBaseline()` | init / ファイル切替 / 外部変更時に呼ぶ。stacks クリア + baseline 設定 |
| `saveSnapshot()` | 編集時に呼ぶ。undoStack 空なら baseline を先に push してから現在状態を push |
| `saveSnapshotDebounced()` | テキスト入力用。500ms debounce で入力後の状態を保存 |

**undo() の skip ロジック:**
undoStack の top が現在の状態と同一の場合、スキップして次のスナップショットを復元する。redo() も同様。

**フロー:**
1. 初期化: `saveBaseline()` → undo disabled
2. 最初の編集: `saveSnapshot()` → undoStack = [baseline, edited] → undo enabled
3. undo: pop edited → restore baseline → undo disabled
4. ファイル切替: `saveBaseline()` → stacks クリア → undo disabled

### 変更ファイル
- `src/webview/outliner.js` — undo/redo アーキテクチャ全面改修

## 修正2: Notes Sidepanel Undo/Redo スコープ分離

### 課題
Notes editor で sidepanel markdown 編集中に Cmd+Z を押すと、markdown と outliner の両方で undo が発火する。

### 原因
editor.js の capture-phase ハンドラが sidepanel の undo を処理した後、outliner.js のグローバルハンドラも発火する。

### 対策
outliner.js のグローバルキーハンドラで `e.defaultPrevented` をチェック。capture handler が処理済みなら skip。

### 変更ファイル
- `src/webview/outliner.js` — グローバルキーハンドラに `e.defaultPrevented` ガード追加

## 修正3: Scope In 検索インジケーター（placeholder 方式）

### 課題
scope in 中に検索がスコープ内に限定されることがUIで明示されていない。

### 対策
DOM要素追加ではなく、scope in 時に検索ボックスの placeholder を「Search in scope」（i18n対応）に変更。scope out で元に戻す。スペースを一切消費しない。

### 変更ファイル
- `src/webview/outliner.js` — `updateScopeSearchIndicator()` で placeholder 切替
- `src/i18n/locales/*.ts` — `outlinerSearchInScope` キー追加（7言語）

## 修正4: Outliner → Markdown リスト形式ペースト

### 課題
Outliner で複数ノードコピー → sidepanel markdown で paste するとプレーンテキストになりリスト形式にならない。

### 対策
Cmd+C/X 時に `navigator.clipboard.write()` で text/plain + text/html（ネスト `<ul>/<li>`）を書き込む。既存の paste ハンドラが Turndown 経由で Markdown リストに変換。

### 変更ファイル
- `src/webview/outliner.js` — `buildSelectedNodesHtml()`, `writeClipboardWithHtml()` 追加
