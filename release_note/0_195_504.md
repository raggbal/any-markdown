# v0.195.504 — Outliner: Scope out時のカーソル位置改善

## 課題

Outlinerのスコープモード（Cmd+]）で特定ノードにスコープインした後、Cmd+Shift+]やパンくずリストのTOPリンクでスコープアウトすると、カーソルが常にアウトライナーの先頭行に移動してしまう。直前までスコープしていたノードに戻りたいのが自然な期待動作。

## 原因

`setScope()` 関数が、スコープ変更後に常に `focusFirstVisibleNode()` を呼び出していたため、スコープイン/アウトに関わらず先頭ノードにカーソルが移動していた。

```javascript
function setScope(scope) {
    currentScope = scope;
    updateBreadcrumb();
    if (searchInput.value.trim()) { executeSearch(); }
    renderTree();
    focusFirstVisibleNode();  // ← 常に先頭行
}
```

## 対策

### 設計

`setScope()` に入る時点で変更前の `currentScope.rootId` を保存し、スコープアウト後のツリーレンダリングで該当ノードの DOM 要素を探してフォーカスする。ノードが見つからない場合（削除済み・検索フィルタで非表示等）は従来通り先頭行にフォールバック。

### 実装

- `setScope()` 冒頭で `previousRootId` を保存
- `renderTree()` 後、`previousRootId` が現在のスコープと異なる場合（= スコープアウトした場合）、`treeEl.querySelector` で該当ノード要素を取得
- `focusNodeElAtStart()` でカーソル移動 + `scrollIntoView({ block: 'nearest' })` で画面内スクロール
- 該当ノードが見つからない場合は `focusFirstVisibleNode()` にフォールバック

### 対象パス

- Cmd+Shift+] → `setScope({ type: 'document' })`
- パンくずTOPリンク → `setScope({ type: 'document' })`
- パンくず中間ノードクリック → `setScope({ type: 'subtree', rootId: ancestorId })`
- 全てのパスで `previousRootId !== scope.rootId` の条件で統一的に処理

### 変更ファイル

- `src/webview/outliner.js`: `setScope()` 関数の修正
