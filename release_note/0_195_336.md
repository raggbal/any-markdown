# v0.195.336 - Perplexityテーマのフォントサイズ・余白・装飾改善

## 課題

Perplexityテーマのフォントサイズが統一的で、要素ごとの最適なサイズ設定がなかった。また余白が大きく、1ページあたりの表示行数が少なかった。h2に視覚的な区切りがなかった。

## 原因

Perplexityテーマのオーバーライドがベーステーマのデフォルト値をそのまま継承しており、テーマ独自のタイポグラフィ最適化が不十分だった。

## 対策

### 設計

ユーザー指定のフォントサイズを基準に全要素のサイズを比例設計:

| 要素 | サイズ |
|---|---|
| p, li (本文) | 16px |
| h1 | 24px |
| h2 | 20px |
| h3 | 18px |
| h4 | 16px |
| h5 | 14px |
| h6 | 13px |
| pre, blockquote, table td/th | 14px |
| inline code | 13px |

余白は全要素で削減:
- body line-height: 1.75 → 1.5
- heading margin: 0.5em → 0.2〜0.4em
- paragraph margin: 0.5em → 0.25em
- list margin: 0.5em → 0.2em, li margin: 0.25em → 0.1em
- pre padding: 16px → 12px, margin: 1em → 0.4em
- table margin: 1em → 0.4em, cell padding: 8px 12px → 5px 10px
- hr margin: 1em → 0.5em

h2装飾: `text-decoration: underline` でテキスト幅のみにアンダーライン表示（`border-bottom`ではなくページ全幅に及ばない）。

### 実装

`src/webview/styles.css` の Perplexity theme overrides セクションを全面改訂。各要素に明示的な`font-size`、`margin`、`padding`を設定。
