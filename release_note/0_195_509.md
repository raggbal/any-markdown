# v0.195.509 — Notes左パネルヘッダー高さ修正

## 課題

Notes機能の左パネル「Outlines」ヘッダーバーと、右側のアウトライナ検索バーの高さが数px合っていなかった。横に並んだ際に視覚的なズレが生じていた。

## 原因

高さの不一致は2つの要因による：

1. **外側パディングの差異**: `.file-panel-header` は `padding: 12px` だが、`.outliner-search-bar` は `padding: 8px`（垂直方向）
2. **内部ボタンの高さの差異**: `.file-panel-btn` は `padding: 4px 6px`（ボタン高さ≈24px）だが、検索バー内の `.outliner-search-input` は `padding: 6px 10px`（入力高さ≈29px）

計算上：
- 検索バー: 8px + 29px + 8px + 1px(border) = 46px
- ヘッダー: 12px + 24px + 12px + 1px(border) = 49px → 3px差

## 対策

### 設計
両要素の外側パディングと内部要素の高さを揃える方針。

### 実装（`src/shared/notes-body-html.js`）

1. `.file-panel-header` のパディングを `12px 12px` → `8px 12px` に変更（検索バーの垂直パディング8pxと統一）
2. `.file-panel-btn` のパディングを `4px 6px` → `6px 8px` に変更（検索インプットの垂直パディング6pxと統一）

修正後：
- 検索バー: 8px + 29px + 8px + 1px = 46px
- ヘッダー: 8px + 28px + 8px + 1px = 45px（SVG 14px + padding 12px + border 2px = 28px）

実質1px以内の差に収束。
