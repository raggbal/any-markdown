# v0.195.398 - サイドパネルTOC（目次）機能

## 課題

サイドパネル（iframe方式）でリンク先の`.md`ファイルを開いた際、ページ内の見出し構造を把握する手段がなかった。長いドキュメントでは特定の見出しに素早くジャンプできず、ナビゲーション性が低かった。

## 原因

サイドパネルにはiframe内のeditor.jsが持つOutline機能（メインエディタのサイドバー）が表示されない設計になっていた。iframe内の`.sidebar`はCSS `display: none !important`で非表示にされており、メインwebview側にもサイドパネル用の目次UIが存在しなかった。

## 対策

### 設計

サイドパネルのヘッダーとiframeの間にTOC（Table of Contents）ナビゲーションパネルを追加。TOCはiframeの**外側**（メインwebview側）に配置し、iframe内エディタとは`scrollToAnchor`メッセージで連携する。

```
┌──────────────────────────────────┐
│ Header (filename + TOC btn + ×)  │
├────────┬─────────────────────────┤
│  TOC   │    iframe (editor.js)   │
│  h1    │                         │
│   h2   │    content...           │
│   h2   │                         │
│  h1    │                         │
└────────┴─────────────────────────┘
```

- TOCデータはeditorProvider.tsでMarkdownからh1/h2を抽出し、`openSidePanel`メッセージに付加
- TOC項目クリック → iframe内の`scrollToAnchor`ハンドラで該当見出しまでスムーズスクロール+ハイライト
- サイドパネル内でMarkdownを編集すると、editメッセージ受信時にTOCを自動再生成
- TOCトグルボタンで表示/非表示を切替（状態はサイドパネルが開いている間維持）

### 実装

1. **editor-body-html.js**: side-panel内にTOCトグルボタン（SVGアイコン）、`side-panel-body`ラッパー（flex横並び）、`side-panel-toc`ナビ要素を追加

2. **styles.css**: `.side-panel-body`（flex横並び）、`.side-panel-toc`（180px幅、h1/h2インデント）、`.side-panel-toc-toggle`（アクティブ状態）、`.side-panel-filename`に`flex:1`追加

3. **editorProvider.ts**: `extractToc()`メソッド追加。Markdownからh1/h2を抽出しアンカースラグを生成。`openSidePanel`メッセージの2箇所（openLink、sidePanelOpenLink）にtocデータを付加

4. **editor.js**:
   - `openSidePanel()`にTOC描画処理を追加
   - `renderSidePanelToc()` — TOCリストのDOM生成
   - `bindSidePanelTocClicks()` — クリックハンドラ（scrollToAnchorメッセージ送信+アクティブ状態更新）
   - `updateSidePanelTocFromMarkdown()` — Markdownからh1/h2を再抽出してTOCを再描画
   - TOCトグルボタンハンドラ
   - iframe editメッセージ受信時にTOC自動更新
