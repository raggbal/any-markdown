# v0.195.512–513 — Notesフォルダ管理機能 + ファイル切替時データ消失バグ修正

## v0.195.512: Notesフォルダ/ツリー管理機能

### 課題
Notesの左パネルはフォルダ直下の.outファイルをフラットに並べるだけで、フォルダによる分類や並び順の指定ができなかった。

### 対策

#### 設計
- 物理フォルダは追加せず、`.note` JSONファイル1つでフォルダ構造を管理
- `.note` のデータモデルはoutlinerのノード管理と同様の `items` マップ + `rootIds` 順序配列
- ディスク上の `.out` ファイルとの自動同期（孤児追加・欠損削除）
- D&DはVSCode webviewの `dataTransfer` 制限を考慮し、module-scope変数で状態管理

#### .note JSONスキーマ
```json
{
  "version": 1,
  "rootIds": ["fileId1", "folderId1"],
  "items": {
    "fileId1": { "type": "file", "id": "fileId1", "title": "..." },
    "folderId1": {
      "type": "folder", "id": "folderId1", "title": "...",
      "childIds": ["fileId2"], "collapsed": false
    }
  }
}
```

#### 実装

| ファイル | 変更 |
|---------|------|
| `notes-file-manager.ts` | `.note` CRUD、syncStructureWithDisk、moveItem（循環参照チェック付き）、createFolder/deleteFolder/renameFolder/toggleFolderCollapsed |
| `notes-message-handler.ts` | 5つの新メッセージ型: notesCreateFolder, notesDeleteFolder, notesRenameFolder, notesToggleFolder, notesMoveItem。共通ヘルパー `sendFileListWithStructure()` |
| `notes-host-bridge.js` | createFolder, deleteFolder, renameFolder, toggleFolder, moveItem メソッド。onFileListChanged に structure 引数追加 |
| `notes-body-html.js` | フォルダ/ツリー/D&D用CSS、フォルダ作成ボタン(SVGアイコン) |
| `notes-file-panel.js` | 全面改修: renderTree再帰レンダリング、フォルダ展開/折畳、D&D（ドロップライン+フォルダハイライト）、フォルダ/ファイル別コンテキストメニュー |
| `notesEditorProvider.ts` | .note構造ロード・配線、ツリー順ファイルオープン、フォルダウォッチャーの構造再同期 |
| `notesWebviewContent.ts` | structure を初期化データとして渡す |

## v0.195.513: ファイル切替時データ消失バグ修正

### 課題1: ファイル切替時に入力中のデータが消える
outlinerで入力して改行せずに左パネルで別ファイルを選択すると、入力内容が消失。

#### 原因
webview側の `scheduleSyncToHost` デバウンス(1秒)がファイル切替前に発火しておらず、未保存データがホストに送信されないまま `Outliner.init(newData)` でモデルが置換される。

#### 対策
- `Outliner` に `flushSync()` メソッドを追加（`syncToHostImmediate()` を公開API化）
- `notes-host-bridge.js` の `openFile()` でファイル切替前に `flushOutlinerSync()` → `Outliner.flushSync()` を呼び、即座にデータを送信

### 課題2: stale syncData によるファイル上書き
ファイル切替後に旧ファイルの `syncData` が到着し、新ファイルを上書きする。

#### 原因
`syncData` メッセージがホスト側で処理される時点で `currentFilePath` は既に新ファイルに変更済み。旧データが新ファイルに書き込まれる。

#### 対策
- `NotesFileManager` に `fileChangeId` カウンターを追加（`openFile()` でインクリメント）
- `notes-host-bridge.js` で `updateData` メッセージから `fileChangeId` を受け取り、`syncData` に付与
- `handleNotesMessage` の `syncData` ハンドラで `fileChangeId` が一致しない場合は無視
