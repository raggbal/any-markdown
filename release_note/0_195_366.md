# v0.195.366 リリースノート

## 課題

長文の段落（インライン書式 `<del>`, `<em>`, `<strong>` を含む）が自動折り返しされた場合、ArrowDown(↓)では折り返しの各視覚行を正しく移動できるが、ArrowUp(↑)では折り返し行が無視されて段落を飛び出してしまう。

## 原因

`isCursorAtFirstLine()` の浮動小数点比較が誤判定を起こしていた。

```javascript
return cursorRect.top < elemRect.top + cursorRect.height;
```

`line-height: 1.6` + `font-size: 16px` の場合、行の高さは 25.6px。しかし 25.6 は IEEE 754 浮動小数点数で正確に表現できない循環小数（`11001.1001100110011...₂`）。

折り返し2行目にカーソルがある場合:
- `cursorRect.top` ≈ `elemRect.top + 25.59999...`（レイアウトエンジンの計算値）
- `elemRect.top + cursorRect.height` ≈ `elemRect.top + 25.60000...001`（JS加算の結果）

加算結果がわずかに大きくなるため `25.59999... < 25.60000...001 → true` となり、2行目を「先頭行」と誤判定。結果として `e.preventDefault()` + `navigateToAdjacentElement(prev, 'up')` が発火し、段落を飛び出す。

ArrowDown の `isCursorAtLastLine()` は減算（`elemRect.bottom - cursorRect.height`）を使うため、浮動小数点の丸め方向が逆になり同じ誤判定が起きにくかった。

## 対策

### 設計

両関数の比較に 2px のトレランスを追加。2px は:
- 浮動小数点誤差（~10⁻¹⁴ px）より圧倒的に大きい
- どんなフォントサイズでも行間距離（最小 8px 程度）より十分小さい

### 実装

**`isCursorAtFirstLine()`** ([editor.js:8216](src/webview/editor.js#L8216)):
```javascript
// Before
return cursorRect.top < elemRect.top + cursorRect.height;
// After
return cursorRect.top < elemRect.top + cursorRect.height - 2;
```

**`isCursorAtLastLine()`** ([editor.js:8250](src/webview/editor.js#L8250)):
```javascript
// Before
return cursorRect.bottom > elemRect.bottom - cursorRect.height;
// After
return cursorRect.bottom > elemRect.bottom - cursorRect.height + 2;
```
