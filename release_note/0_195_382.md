# v0.195.382: Outlineパネルデザイン改善 + スクロール不具合修正

## 変更概要

### 1. Outline/Editor境界線の微調整

**課題:** Outline(サイドバー)とEditorの境界線が `--border-color` で描画されており、視覚的に強すぎた。

**対策:**
- 各テーマに `--sidebar-border` CSS変数を新設
- ライト系テーマ: `--sidebar-bg` をわずかに暗くした色
- ダーク系テーマ: `--sidebar-bg` をわずかに明るくした色
- 当初 `color-mix(in srgb, var(--sidebar-bg) 96%, black)` を試みたが、ダーク系テーマでは黒を混ぜると見えなくなるため、テーマごとに明示的な色を定義する方式に変更

| テーマ | --sidebar-bg | --sidebar-border |
|---|---|---|
| github | #f6f8fa | #eaecef |
| sepia | #f3efe4 | #e7e3d8 |
| night | #16161e | #1e1e28 |
| dark | #252526 | #2e2e2f |
| minimal | #f5f5f5 | #e9e9e9 |
| perplexity | #F2F1EB | #E6E5DF |

### 2. Outlineヘッダ下線の非表示化

**課題:** `.sidebar-header` の `border-bottom` が目立っていた。

**対策:** `border-bottom` の色を `var(--sidebar-bg)` に変更し、背景と同化させて視覚的に非表示化。

### 3. Outlineパディング調整

**課題:** Outline項目の `padding-left` が 16px スタートで狭かった。sidebar-header のパディングも不足。

**対策:**
- `.sidebar-header` padding: 16px → 24px
- `.outline-item[data-level]` padding-left: 24px スタート、12px 刻み (24/36/48/60/72)

### 4. Outline目次クリック後のスクロール不具合修正

**課題:** Outline目次をクリックすると `scrollIntoView()` が発火し、ターゲットの見出しだけでなく親コンテナ（body等、`overflow: hidden`）もスクロールされてしまう。`overflow: hidden` のためユーザーが手動でスクロールを戻せなくなる。

**原因:** `Element.scrollIntoView()` はDOMツリー上の全スクロール可能祖先をスクロールする仕様。`overflow: hidden` な要素でもスクロール位置が変わるが、ユーザーのマウスホイールでは戻せない。

**対策:**
- `scrollIntoView()` を廃止し、`.editor-wrapper` の `scrollTo()` を直接呼び出す方式に変更
- `heading.getBoundingClientRect()` と `wrapper.getBoundingClientRect()` の差分で正確なスクロール量を計算
- 対象箇所: Outline目次クリック（`updateOutline`）、アンカーリンクジャンプの2箇所
