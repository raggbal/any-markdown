## 1. Scope-in時のカーソル位置改善

### 課題
Scope-inすると、最初の子ノードの先頭にカーソルが置かれていた。スコープヘッダー（スコープ対象ノード）のテキスト末尾にカーソルを置くのが自然。

### 原因
`setScope()` が `renderTree()` 後に `focusFirstVisibleNode()` を呼んでおり、最初の `.outliner-node`（スコープヘッダー）の先頭にカーソルが配置されていた。

### 対策（設計）
`setScope()` 末尾で、subtreeスコープの場合に `focusNode(currentScope.rootId)` を呼ぶ。`focusNode()` は `setCursorToEnd()` を使用するため末尾にカーソルが配置される。

### 対策（実装）
`outliner.js` の `setScope()` 末尾を条件分岐に変更:
```javascript
if (currentScope.type === 'subtree' && currentScope.rootId) {
    focusNode(currentScope.rootId);
} else {
    focusFirstVisibleNode();
}
```

## 2. Daily Notes: 空ノード自動追加の削除

### 課題
v0.195.523でスコープ空状態のEnter/クリックによる子ノード追加UIが実装されたため、Daily Notesでの空ノード自動追加が不要になった。

### 原因
`notes-message-handler.ts` の3つのDaily Notesハンドラ（`notesOpenDailyNotes`, `notesNavigateDailyNotes`, `notesNavigateToDate`）で `dayNode.children.length === 0` の場合に `addNodeToData()` で空ノードを追加していた。

### 対策
3箇所の空ノード自動追加ブロックを削除。空スコープ状態ではv0.195.523のUIでユーザーが自分でノードを追加できる。

## 3. Shift+Tabデインデントのスコープ境界制限

### 課題
スコープ対象ノードの直接の子でShift+Tabを押すと、ノードがスコープ外（スコープ対象ノードの兄弟）に移動し、画面から消えてしまう。

### 原因
`handleShiftTab()` → `model.outdentNode()` にスコープ境界のガードがなかった。

### 対策（設計）
UI層（`handleNodeKeydown`）でスコープルートの直接の子の場合にデインデントをブロック。モデル層は変更しない。

### 対策（実装）
`case 'Tab':` 内のShift+Tab分岐にガード追加:
```javascript
if (currentScope.type === 'subtree' && currentScope.rootId && node.parentId === currentScope.rootId) {
    break;
}
```
孫以降のノードは通常通りデインデント可能。`saveSnapshot()` をガード後に移動し、undo履歴を汚さない。

## 4. テキスト全選択状態でのBackspace動作修正

### 課題
ノードのテキストをダブルクリック等で全選択してBackspaceを押すと、テキストが削除されるべきところ、行先頭でのBackspace（前行マージ）と同じ挙動になっていた。

### 原因
`getCursorOffset()` は `range.startContainer` のオフセットを返すため、全選択時は `offset === 0` → `isAtStart === true` → `handleBackspaceAtStart()` が発動。`selection.isCollapsed` をチェックしていなかった。

### 対策（設計）
選択範囲がある（`!selection.isCollapsed`）場合、カスタム処理をスキップしブラウザのデフォルト動作に任せる。`input`イベントハンドラで `getPlainText()` → `model.updateText()` が自動同期。

### 対策（実装）
`case 'Backspace':` 冒頭に選択範囲チェック追加:
```javascript
var bsSel = window.getSelection();
var hasSelection = bsSel && !bsSel.isCollapsed;
if (hasSelection) {
    saveSnapshot();
    break;  // ブラウザのデフォルト動作に委ねる
}
```
スコープヘッダーガードも `!hasSelection` 条件を追加し、全選択Backspaceを許可。
