# v0.195.536 リリースノート

## 修正内容

### 1. Outliner/Notes CSS変数の不整合修正

**課題:** `outliner.css` の `:root` に未定義のCSS変数が複数あり、参照箇所でフォールバック値に依存していた。変数名の不一致（`--outliner-subtext` vs `--outliner-subtext-color`）もあった。

**原因:** `notes-body-html.js` や `outliner.css` の各セレクタで `var(--outliner-hover)`, `var(--outliner-text)`, `var(--outliner-active)`, `var(--outliner-subtext)` を参照していたが、`:root` および各テーマセレクタに定義がなかった。

**対策:**
- **設計:** `:root` にデフォルト値を追加し、全7テーマ（github, sepia, night, dark, minimal, things, perplexity）にテーマ適合値を定義。
- **実装:** 以下の変数を `:root` + 全テーマに追加:
  - `--outliner-text` — テキスト色
  - `--outliner-hover` — ホバー背景色
  - `--outliner-active` — アクティブ背景色
  - `--outliner-subtext` — サブテキスト色
  - `--outliner-subtext-color` — サブテキスト色（別名）

### 2. ハードコード色のCSS変数化

**課題:** `outliner.css` 内に `#d00`（削除ボタン）、`#fff`（アクティブタグ、日付ピッカー選択）がハードコードされており、テーマに応じた色の変更ができなかった。

**原因:** 初期実装時にテーマ対応を想定せず直接色値を記述していた。

**対策:**
- **設計:** 新CSS変数 `--outliner-error-color`（エラー/削除色）と `--outliner-accent-fg`（アクセント前景色）を導入。
- **実装:**
  - `.pinned-tag-delete:hover` の `color: #d00` → `var(--outliner-error-color, #d00)`
  - `.outliner-pinned-tag-btn.is-active` の `color: #fff` → `var(--outliner-accent-fg, #fff)`
  - `.outliner-daily-picker-day.selected` の `color: #fff` → `var(--outliner-accent-fg, #fff)`
  - 全7テーマにテーマ適合のエラー色を定義

### 3. Electron i18n英語フォールバック追加

**課題:** Electronアプリで、リクエストされたロケールファイルが見つからない場合、空オブジェクト `{}` を返していた。UI文字列が全て空白になるリスクがあった。

**原因:** `getI18nMessages()` のフォールバックチェーンに英語ロケールへの遡及がなかった（VSCode版は `messages.ts` で英語フォールバックを持つが、Electron版は独自実装）。

**対策:**
- **設計:** `return {}` の前に英語ロケール (`en.js`) のロードを試行する。英語も見つからない場合のみ従来通り `{}` を返す。
- **実装:** `electron/src/main.ts` の `getI18nMessages()` に英語フォールバックブロックを追加。
