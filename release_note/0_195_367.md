# v0.195.367 リリースノート

## 課題

長文段落の直下の要素（コードブロック、テーブル等）から↑キーで段落に入ると、最終行ではなく必ず先頭行にカーソルが移動してしまう。

## 原因

`setCursorToLastLineStartByDOM()` がDOMレベルの改行（`<br>` と `\n` テキストノード）のみを行区切りとして検出していた。CSSの自動折り返し（soft wrap）で複数の視覚行に分かれた段落には DOM改行が存在しないため、`lineBreaks.length === 0` → `setCursorToStart(element)` で先頭にカーソルが置かれていた。

## 対策

### 設計

`lineBreaks.length === 0` の場合に `getBoundingClientRect()` を使って視覚行を検出する:

1. TreeWalkerで全テキストノードを収集し、フラットなオフセットマップを構築
2. 最終文字と先頭文字のY座標を比較 → 差が2px未満なら単一行（`setCursorToStart`で正しい）
3. 複数行の場合、バイナリサーチで「最終視覚行のY座標と同じY座標を持つ最初のオフセット」を特定
4. そのオフセットに対応するテキストノード・位置にカーソルを配置

### 実装

[editor.js](src/webview/editor.js) の `setCursorToLastLineStartByDOM()` 内、`lineBreaks.length === 0` ブランチを修正。

バイナリサーチは `O(log N)` イテレーション（N=総文字数）で、各イテレーションで1文字分のRangeを作成して `getBoundingClientRect()` でY座標を取得。典型的な段落（200文字）で約8回のイテレーション。
