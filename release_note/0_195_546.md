# v0.195.546 リリースノート

## Outliner エディタ デザインリフレッシュ

### 課題
Outliner editorのデザインが古く感じられる問題:
1. エディタ左右に余白がなく、コンテンツがぎっしり詰まっている
2. ツールバー各セクション（検索バー、固定タグバー、パンくず）の間に仕切り線が多く、うるさい
3. 検索バーが常に100%幅で、コンパクトさに欠ける

### 原因
- 左右パディングが16pxと狭く、現代的なUIの余白基準（32px程度）を満たしていなかった
- 検索バー、固定タグバー、パンくずそれぞれに`border-bottom`が設定されており、視覚的に重い
- ツールバーボタンにも`border`が設定されており、要素が多い印象
- 検索入力欄が`flex: 1`で常に全幅を占有していた

### 対策

#### 設計
- CSS変数 `--outliner-gutter` (32px) を導入し、全体の余白を一元管理
- ツールバー間の仕切り線をすべて除去し、アウトライン本体の上にだけ1本線を残す
- 検索バーは固定幅250pxで表示し、フォーカス時にflex展開するインタラクション
- ボタンのborderを除去し、hover時は背景色変化のみのフラットデザインに

#### 実装 (outliner.css)

**余白の統一:**
- `.outliner-container` に `--outliner-gutter: 32px` を定義
- `.outliner-search-bar`, `.outliner-pinned-nav-bar`, `.outliner-breadcrumb`, `.outliner-page-title` のpaddingを `var(--outliner-gutter)` に統一
- `.outliner-node` に `margin-left: var(--outliner-gutter)` と `padding-right: var(--outliner-gutter)` を追加
- メニュードロップダウン、デイリーピッカー、フォーカス祖先パンくずのposition値も更新

**仕切り線の除去:**
- `.outliner-search-bar` の `border-bottom` を削除
- `.outliner-pinned-nav-bar` の `border-bottom` を削除
- `.outliner-breadcrumb` の `border-bottom` を削除
- `.outliner-tree` に `border-top: 1px solid var(--outliner-border)` を追加（唯一の仕切り線）

**ボタンのフラット化:**
- `.outliner-search-mode-toggle`, ナビボタン, undo/redo, メニューボタンの `border` を `none` に
- hover時のスタイルを `border-color` 変更から `background: var(--outliner-hover-bg)` に変更
- デフォルト opacity を 0.7 → 0.5 に下げ、よりミニマルに

**検索バーの幅制御:**
- `.outliner-search-input-wrapper` を `width: 250px; flex-shrink: 0` に変更
- `:focus-within` 疑似クラスで `flex: 1; width: auto` に展開
- `transition: width 0.2s ease` でスムーズなアニメーション
- 検索入力欄の `border` を transparent に、`border-radius` を 6px に
