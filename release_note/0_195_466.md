# v0.195.466 リリースノート

## 修正内容: block patternのregexが既存テキスト直結時にマッチしない問題

### 課題

markdownモードで、既にテキストが入力されている行の先頭にinput patternを入力してspaceを押した場合、block変換が動作しない。

例:
- `<h1>heading</h1>` の行頭で `##` + space → h2に変換されるべきだが、されない
- `<li>aaaa</li>` (bullet) の行頭で `[ ]` + space → タスクリストに変換されるべきだが、されない

### 原因

v0.195.465でspace処理をpreventDefault方式に変更した際、block patternのregex 8箇所が「spaceなしでパターンとテキストが直結するケース」を想定していなかった。

全regexが `(?: (.*))?$` パターンを使用しており、これは `]` の後に **space + テキスト** を期待する。preventDefaultによりspaceが挿入されないため、`[ ]aaaa` のようにパターンとテキストが直結すると、spaceがないためグループがマッチせず、残りの `aaaa` が `$` と一致できずマッチ失敗となる。

```
# マッチしないケース
"##heading"  → /^(#{1,6})(?: (.*))?$/  → group skipped → $ fails (aaaa remains)
"[ ]aaaa"    → /^\[([ xX])\](?: (.*))?$/ → group skipped → $ fails
```

### 対策

#### 設計
全8箇所のregexで `(?: (.*))?$` を `\s?(.*)$` に変更。`\s?` でspaceをオプショナルにし、`(.*)` で残りテキスト（空でもOK）をキャプチャする。

#### 実装

変更対象8箇所:

| 箇所 | 変更前 | 変更後 |
|------|--------|--------|
| リスト内task変換 (L4145) | `^\[([ xX])\](?: (.*))?$` | `^\[([ xX])\]\s?(.*)$` |
| リスト内bullet変換 (L4188) | `^[-*+](?: (.*))?$` | `^[-*+]\s?(.*)$` |
| リスト内ordered変換 (L4216) | `^(\d+)\.(?: (.*))?$` | `^(\d+)\.\s?(.*)$` |
| heading (L4251) | `^(#{1,6})(?: (.*))?$` | `^(#{1,6})\s?(.*)$` |
| task list block (L4277) | `^[-*+] \[([ xX])\](?: (.*))?$` | `^[-*+] \[([ xX])\]\s?(.*)$` |
| bullet list block (L4321) | `^[-*+](?: (.*))?$` | `^[-*+]\s?(.*)$` |
| ordered list block (L4364) | `^(\d+)\.(?: (.*))?$` | `^(\d+)\.\s?(.*)$` |
| blockquote (L4399) | `^>(?: (.*))?$` | `^>\s?(.*)$` |

キャプチャグループ番号は変更なし。`existingText` が `undefined` → `""` に変わるが、全箇所 `|| ''` で処理済みのため影響なし。
