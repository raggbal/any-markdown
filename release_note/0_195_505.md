# v0.195.505 — Notes 機能（VSCode Activity Bar）

## 概要

Electron版の outliner フォルダ管理機能を「Notes」として共通化し、VSCode の Activity Bar から利用可能にした。

## 課題

- Electron版には .out ファイル群を束ねるフォルダ管理機能（左パネル + outliner エディタ）があったが、VSCode拡張では利用できなかった
- .out ファイルを1つずつ開く運用しかできず、複数アウトラインの管理が不便だった

## 対策

### 設計

- ビジネスロジック（ファイルCRUD、メッセージルーティング、UI HTML/JS）を `src/shared/` に配置して完全共有
- 各プラットフォームは sender/platform の接続のみを行う薄いラッパー
- 既存の .out 単体エディタ、.md エディタには一切変更なし（新規ファイルのみ）

### 実装

**新規ファイル (9個):**

| ファイル | 役割 |
|---------|------|
| `src/shared/notes-file-manager.ts` | .outファイルCRUD、デバウンス保存、pageDir解決（純粋Node.js） |
| `src/shared/notes-message-handler.ts` | webviewメッセージルーティング共通ロジック |
| `src/shared/notes-body-html.js` | 左パネルCSS+HTML生成 |
| `src/shared/notes-file-panel.js` | 左パネルUIコントローラ（ブラウザ側） |
| `src/shared/notes-host-bridge.js` | VSCode webview用ブリッジ（outliner.js + file-panel用） |
| `src/notesFolderProvider.ts` | Activity Bar TreeDataProvider（フォルダ一覧、globalState永続化） |
| `src/notesEditorProvider.ts` | WebviewPanel管理（単一パネル、メッセージ委譲、フォルダ監視） |
| `src/notesWebviewContent.ts` | Notes用HTML生成（左パネル + outliner + サイドパネル） |
| `media/notes-icon.svg` | Activity Barアイコン |

**変更ファイル (3個):**

| ファイル | 変更内容 |
|---------|---------|
| `package.json` | viewsContainers, views, commands (add/remove/openNotesFolder), menus追加 |
| `src/extension.ts` | NotesFolderProvider + NotesEditorProvider + コマンド3つ登録 |
| `src/outlinerWebviewContent.ts` | CSP に `frame-src blob:;` 追加（サイドパネルiframe用） |

### 機能

- Activity Bar に Notes アイコン表示
- フォルダ追加/削除（OS ダイアログ）、一覧永続化
- フォルダクリックで WebviewPanel 開く
- 左パネル: .out ファイル一覧（タイトル表示）、追加/削除/リネーム/切替
- メインエリア: outliner エディタ（既存 outliner.js そのまま）
- 右サイドパネル: ページ表示（既存 EditorInstance + SidePanelManager）
- デバウンス保存（1秒）、ファイル切替時フラッシュ
- フォルダ監視（外部から .out ファイル追加/削除時に左パネル更新）
- テーマ/フォントサイズ変更時の動的更新
- 同時に開けるパネルは1つのみ

### テスト結果

- 既存テスト 653 passed（失敗6件は既存の horizontal-rule-backspace + perplexity-highlight で Notes とは無関係）
