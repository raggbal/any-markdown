# v0.195.540 リリースノート

## Outliner コンテキストメニューの改善

### 課題
Outliner editorの右クリックコンテキストメニューに以下の問題があった:
1. メニュー項目名が不明瞭（「ページ解除」「削除」）
2. ノード追加（兄弟/子）やインデント操作がメニューになく、ショートカットを知らないと使えなかった
3. メニュー項目にショートカットキーの表示がなく、学習性が低かった
4. メニュー項目の並び順が操作の論理グループに沿っていなかった

### 対策

#### 設計
- メニュー項目をリネーム: 「ページ解除」→「ページ削除」、「削除」→「ノードを削除」
- 「ページ削除」を「ページを開く」の直下に移動（ページ操作をグループ化）
- 新規メニュー項目追加: 「兄弟ノードを追加」「子ノードを追加」「インデント」「デインデント」
- 各メニュー項目にキーボードショートカットをタグ風デザインで表示
- プラットフォーム検出（Mac: Cmd/Option、Win: Ctrl/Alt）でショートカット表記を切替

#### 実装

**outliner.js:**
- `isMacPlatform` / `modLabel` 変数追加（プラットフォーム検出）
- `addMenuItem(parent, text, handler, shortcut)` に第4引数 `shortcut` を追加
- メニュー項目を `<span class="context-menu-label">` + `<span class="context-menu-shortcut">` の2要素構成に変更
- `showContextMenu` を新しいメニュー構成で再構築:
  1. ページ操作（Open Page / Delete Page / Make Page）
  2. ノード追加（Add Sibling Node / Add Child Node）
  3. インデント（Indent / Dedent）
  4. チェックボックス操作
  5. サブテキスト操作
  6. スコープ操作
  7. 移動（Move Up / Move Down）
  8. ノード削除（Delete Node）

**outliner.css:**
- `.outliner-context-menu-item` に `display: flex; justify-content: space-between` 追加
- `.context-menu-label` スタイル追加
- `.context-menu-shortcut` タグ風デザイン追加（背景色、border-radius、小フォント）

**i18n（全7言語）:**
- 新規キー追加: `outlinerDeletePage`, `outlinerAddSibling`, `outlinerAddChild`, `outlinerIndent`, `outlinerDedent`, `outlinerDeleteNode`
- 既存キー `outlinerRemovePage`, `outlinerDelete` は互換性のため残置
