# v0.195.539 リリースノート

## 1. Copy Path ボタンの追加

### 課題
編集中のファイルパスをクリップボードにコピーする手段がなかった。

### 対策

#### 設計
- **Standalone editor**: ツールバー右端（ソースモード切替の右）に `copyPath` ボタンを追加
- **Sidepanel editor**: ヘッダーの「Open in new tab」と「Expand」の間に配置
- HostBridge パターンに従い、webview → host メッセージ → ホスト側でクリップボード書き込み

#### 実装
- `host-bridge.ts`: `copyFilePath()` をインターフェースに追加
- `vscode-host-bridge.js` / `test-host-bridge.js`: 実装追加
- `editorProvider.ts`: `vscode.env.clipboard.writeText(document.uri.fsPath)` でコピー
- `editor-body-html.js`: toolbar に `<button data-action="copyPath">` 追加
- `editor.js`: `case 'copyPath'` ハンドラ、SidePanelHostBridge スタブ追加
- `editor-utils.js`: copyPath 用 SVG アイコン追加
- i18n: 全7言語に `copyPath` キー追加
- Electron: `preload.ts/js` に IPC 送信、`main.ts/js` に `copy-file-path` ハンドラ追加
- `outliner.js`: サイドパネル copy path クリックハンドラ追加

## 2. サイドパネルHTML共通化

### 課題
サイドパネルHTMLが4箇所に重複定義されていた:
1. `src/shared/editor-body-html.js`（Markdown editor）
2. `src/outlinerWebviewContent.ts`（Outliner editor）
3. `src/notesWebviewContent.ts`（Notes editor）
4. `electron/src/html-generator.ts`（Electron版）

copy-path ボタン追加時に outliner/notes に追加漏れが発生し、Outliner/Notes からサイドパネルを開いた場合にボタンが表示されなかった。

### 原因
各エディタの WebviewContent で同一のサイドパネルHTMLをコピペしており、変更が同期されなかった。

### 対策

#### 設計
`editor-body-html.js` に `generateSidePanelHtml(messages)` 関数を新設し、全エディタからこの関数を呼ぶ形に統一。既存の `notes-body-html.js` → `generateNotesFilePanelHtml()` と同じ共通化パターンを踏襲。

#### 実装
- `editor-body-html.js`: `generateSidePanelHtml(messages)` を新設・export。`generateEditorBodyHtml` 内のインラインHTMLを関数呼び出しに置換
- `outlinerWebviewContent.ts`: 50行のインラインHTML → `require()` + 関数呼び出し2行
- `notesWebviewContent.ts`: 同上
- `electron/html-generator.ts`: 同上。resize-handle も追加（今まで欠落していた）
- メッセージ補間: `msg.key || 'Fallback'` パターンに統一

#### 結果
サイドパネルHTML定義箇所: **4箇所 → 1箇所**
