# v0.195.585 リリースノート

## ESCキーのイベント伝播修正 — Action Panel / Command Palette / 検索ボックス

### 課題

Outlinerからサイドパネルで.mdファイルを開き、Action Panelを開いた状態でESCキーを押すと、Action Panelだけでなくサイドパネルも同時に閉じてしまう。本来はAction Panelのみが閉じ、サイドパネルは開いたまま維持されるべき。

同様の問題がCommand Palette、検索ボックス（Ctrl+F）でも発生する可能性があった。

### 原因

ESCキーハンドラのイベント伝播制御が不足していた。

- **Action Panel** (editor.js): capture-phaseのメニューESCハンドラ、pathInput/linkNameのinput-levelESCハンドラが `e.preventDefault()` のみ呼び、`e.stopPropagation()` を呼んでいなかった
- **Command Palette** (editor.js): 同様に `e.stopPropagation()` が欠落
- **検索ボックス** (editor.js): searchInput/replaceInputのESCハンドラも同様

これにより、ESCイベントがdocumentレベルのサイドパネルESCハンドラ（outliner.js / editor.js 両方）まで伝播し、サイドパネルが閉じられていた。

### 対策

#### 設計方針

ESCキーは「最も手前のモーダル的UIを閉じる」という一般的なUX原則に従い、イベント伝播を適切に制御する。ESCを処理したハンドラが `stopPropagation()` を呼ぶことで、背後のUI層にイベントが到達しないようにする。加えて、サイドパネルESCハンドラ側にもモーダルUI表示チェックを追加し、二重の安全策とする。

#### 実装

**editor.js (6箇所)**:
1. Action Panel capture-phase メニューステップESC: `e.stopPropagation()` 追加
2. Action Panel pathInput ESC: `e.stopPropagation()` 追加
3. Action Panel linkName確定ESC: `e.stopPropagation()` 追加
4. Command Palette ESC: `e.stopPropagation()` 追加
5. 検索ボックス searchInput ESC: `e.stopPropagation()` 追加
6. 検索ボックス replaceInput ESC: `e.stopPropagation()` 追加

**editor.js サイドパネルESCハンドラ (安全策)**:
- `actionPanelVisible || commandPaletteVisible` チェックを追加。いずれかが表示中ならサイドパネルを閉じない

**outliner.js サイドパネルESCハンドラ (安全策)**:
- DOMクエリで `.action-panel` / `.command-palette` の表示状態をチェック。いずれかが表示中（`style.display !== 'none'`）ならサイドパネルを閉じない
- outliner.jsからはeditor.jsのスコープ変数にアクセスできないため、DOMベースの検査を使用
