# v0.195.483 — 画像ペースト/D&D時の段落分離

## 課題

空の段落で画像をCmd+Vで連続ペーストすると、すべての画像が同一の`<p>`内にインライン挿入され、Markdownでは1行に`![](...)![](...)![](...)`と連結されてしまう。見た目は画像ごとに改行されているように見えるが、実際は1つの段落内に並んでいる。

## 原因

`insertImageHtml`ハンドラが画像挿入後にカーソルを画像直後に置くだけで、新しい段落を作成していなかった。そのため次のペーストも同じ`<p>`内に挿入される。

## 対策

### 設計

画像挿入後、親ブロック（`<p>`/`<div>`）内にテキストノードがない（画像のみ）場合、新しい空の`<p>`を作成してカーソルを移動する。これにより次のペースト/D&Dは新段落に入る。

### 実装

`src/webview/editor.js`** — **`insertImageHtml`**ハンドラ:**

1. `img.closest('p, div')`で親ブロックを取得
2. `parentBlock.closest('.editor') === editor`でエディタ直下かチェック
3. `TreeWalker(SHOW_TEXT)`で親ブロック内の実テキストノードを探索
4. テキストなし（`hasRealText === false`）の場合:
  - 親ブロック内の残存を全削除（ブロッククローザーBRによる余分な隙間防止）
  - 新しい`<p></p>`を親ブロック直後に挿入
  - カーソルを新段落に移動
5. テキストありの場合: 従来通りカーソルを画像直後に配置（インライン挿入）

### デバッグで発見した問題

初回実装で`parentBlock.closest('#editor')`を使用していたが、エディタ要素は`id="editor"`ではなく`class="editor"`だったため、条件が常にfalseとなり新段落が作成されなかった。`.closest('.editor')`に修正して解決。

### テスト結果

- 空段落でCmd+V連続ペースト: 各画像が個別の`<p>`に分離される
- テキストのある段落でペースト: 従来通りインライン挿入
- D&D: 正常に1回だけ挿入される

---

現在の「画像挿入後に新しい `<p>` を作る」条件は：

1. **画像の親ブロックが **`<p>`** または **`<div>` — `img.closest('p, div')`
2. **そのブロックが **`.editor`** の直下ツリーにある** — `parentBlock.closest('.editor') === editor`
3. **そのブロック内にテキストノードがない（画像のみ）** — `TreeWalker` で実テキストを探し、`hasRealText === false`

つまり：

| カーソル位置 | 動作 |
| --- | --- |
| 空の `<p>` にペースト | 新 `<p>` 作成 + カーソル移動 |
| テキストのある `<p>` の途中にペースト | インライン挿入（カーソルは画像直後） |
| `<li>` 内にペースト | インライン挿入（`<li>` は `<p>/<div>` ではないので対象外） |
| `<blockquote>` 内の `<p>` にペースト | 条件1-3を満たせば新 `<p>` 作成 |
| `<td>/<th>` 内にペースト | インライン挿入（テーブルセルは対象外） |
