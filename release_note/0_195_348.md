# v0.195.348 — 空エディタのプレースホルダーがペースト後に消えない問題を修正

## 課題

空のエディタ（新規ファイル等）でCMD+Vでテキストをペーストすると、プレースホルダーテキスト「Start typing...」が消えずに残り続ける。キーボード入力では正常に消える。

## 原因

ペーストハンドラ（`editor.js:12780`）が `e.preventDefault()` でブラウザのデフォルト動作を抑止しているため、`input` イベントが発火しない。`updatePlaceholder()` は `input` イベントハンドラ内でのみ呼ばれていたため、ペースト後にプレースホルダー状態が更新されなかった。

影響範囲:
- インラインテキストペースト
- ブロック要素ペースト
- URLペースト（自動リンク化）
- コードブロック/引用/テーブルセル内ペースト

## 対策

### 設計

`updatePlaceholder()` を個別のペーストパスに追加するのではなく、`syncMarkdownSync()` と `syncMarkdown()` の中に組み込む方針を採用。これらは構造変更後に必ず呼ばれる同期関数であるため、ペーストに限らず全ての構造変更パスでプレースホルダー状態が正しく更新される。

### 実装

- `syncMarkdownSync()` (editor.js:5123) — 末尾に `updatePlaceholder()` 追加
- `syncMarkdown()` (editor.js:5110) — `requestAnimationFrame` コールバック内に `updatePlaceholder()` 追加

### テスト

`test/specs/placeholder-paste.spec.ts` を新規作成（3テストケース）:
1. 空エディタにインラインテキストをペースト → プレースホルダーが消える
2. 空エディタにブロック要素をペースト → プレースホルダーが消える
3. 空エディタにURLをペースト → プレースホルダーが消える

全557テスト合格（既存テストへの影響なし）。
