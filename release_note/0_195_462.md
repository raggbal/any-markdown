# v0.195.462 - タブ後スペースのインデント誤カウント修正

## 課題

テキスト先頭にスペースがあるノード（例: ` bbb`）を含む複数行をコピー＆ペーストすると、ノードの順序が入れ替わる。

### 再現例

```
- aaa
  -  bbb   ← 先頭スペースあり
  - ccc
```

ペースト結果:
```
- aaa
  - ccc
-  bbb     ← 順序が入れ替わり、インデントも壊れる
```

## 原因

インデント解析で `\t bbb` を `タブ(1レベル) + スペース(1レベル) = 2レベル` と誤計算。実際はスペースはテキスト `" bbb"` の一部。

コピー形式はタブ区切りなので、タブ後のスペースはインデントではなくテキストの一部。

## 対策

`pasteNodesFromText` のインデント解析ロジックに `sawTab` フラグを追加：

- タブが見つかった後のスペースはインデントとして数えない（テキストの一部）
- スペースのみで始まる行（外部ペースト）は従来通りスペースをインデントとして計算

```javascript
var sawTab = false;
while (j < line.length) {
    if (line[j] === '\t') { tabs++; j++; sawTab = true; }
    else if (line[j] === ' ' && !sawTab) {
        // スペースのみの行: 外部ペースト対応
        ...
    }
    else { break; }
}
```
