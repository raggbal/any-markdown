# v0.195.388 リリースノート

## 修正内容: Perplexity/ThingsテーマのfontSize設定無視を修正

### 課題
Perplexity/ThingsテーマでユーザーのfontSize設定が反映されない。他のテーマでは正常に動作する。

### 原因
両テーマがbodyの`font-size`をハードコード(`16px`/`15px`)しており、CSS変数`var(--font-size)`を使用していなかった。さらに見出し・コードブロック・引用・テーブル等の要素も固定`px`値でfont-sizeを指定していたため、ユーザー設定に連動しなかった。

### 対策

#### 設計
- bodyの`font-size`を`var(--font-size)`に変更（Thingsは`calc(var(--font-size) - 1px)`でデフォルト16px時に15px基準を維持）
- 各要素の固定`px`値を`em`単位に変換（基準font-sizeに対する比率）

#### 実装
**Perplexityテーマ（基準16px）:**
- body: `16px` → `var(--font-size)`
- h1: `28px` → `1.75em`, h2: `20px` → `1.25em`, h3: `18px` → `1.125em`
- h4: `16px` → `1em`, h5: `14px` → `0.875em`, h6: `13px` → `0.8125em`
- pre/blockquote/th/td: `14px` → `0.875em`, code: `13px` → `0.8125em`

**Thingsテーマ（基準15px）:**
- body: `15px` → `calc(var(--font-size) - 1px)`
- h1: `26px` → `1.733em`, h2: `19px` → `1.267em`, h3: `17px` → `1.133em`
- h4: `15px` → `1em`, h5: `14px` → `0.933em`, h6: `13px` → `0.867em`
- pre/code: `13px` → `0.867em`, blockquote/th/td: `14px` → `0.933em`
