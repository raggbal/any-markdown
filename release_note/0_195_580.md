# v0.195.580 リリースノート

## fractal.fontSize デフォルト変更 & 相対サイズ化

### 課題

1. **デフォルトフォントサイズが大きい**: `fractal.fontSize` のデフォルトが 16px で、14px に変更したい
2. **outliner.css が `fractal.fontSize` を無視するバグ**: `--outliner-font-size: 16px` がハードコードされており、ユーザーが設定を変更しても Outliner/Notes エディタに反映されなかった
3. **コンテンツ領域のサイズが font-size と連動しない**: ノードの行高さ(24px)、最小高さ(28px)、サブテキスト(12px)等がハードコードされており、font-size 変更時にレイアウトバランスが崩れていた
4. **ビルドスクリプトの既存バグ**: `replace('__FONT_SIZE__', '16px')` で `px` 付きの値を渡していたが、CSS側が `__FONT_SIZE__px` 形式のため `16pxpx` になる潜在バグがあった

### 原因

- `outlinerWebviewContent.ts`, `notesWebviewContent.ts`, `electron/html-generator.ts` で `outliner.css` 読み込み時に `__FONT_SIZE__` 置換処理が欠落
- outliner.css のコンテンツ領域要素が px ハードコードで、CSS変数や em 相対指定を使っていなかった
- styles.css の `.editor pre` と `.source-editor` も `font-size: 14px` 固定で、font-size 変更に追従しなかった

### 対策

#### 設計方針: 「コンテンツ領域は相対、UIクロムは固定」

- コンテンツ領域（ユーザーが読み書きするテキスト）→ em ベースの相対指定
- UIクロム（ツールバー、検索、メニュー等）→ 固定 px 維持

#### 実装

**デフォルト値変更 (16 → 14)**:
- `package.json`: `"default": 14`
- `webviewContent.ts`: `fontSize: config?.fontSize ?? 14`
- `outlinerProvider.ts`: `config.get<number>('fontSize', 14)`
- `notesEditorProvider.ts`: `config.get<number>('fontSize', 14)`

**outliner.css 動的置換（バグ修正）**:
- `outliner.css`: `--outliner-font-size: 16px` → `--outliner-font-size: __FONT_SIZE__px`
- `outlinerWebviewContent.ts`: `.replace('__FONT_SIZE__', String(config.fontSize))` 追加
- `notesWebviewContent.ts`: 同上
- `electron/html-generator.ts`: 同上

**outliner.css コンテンツ領域の相対化**:
- `--outliner-indent`: `24px` → `1.5em`
- `.outliner-node min-height`: `28px` → `1.75em`
- `.outliner-bullet height`: `28px` → `1.75em`
- `.outliner-scope-btn height`: `28px` → `1.75em`
- `.outliner-page-icon height`: `28px` → `1.75em`
- `.outliner-checkbox height`: `28px` → `1.75em`
- `.outliner-text min-height`: `24px` → `1.5em`
- `.outliner-text line-height`: `24px` → `1.5`（単位なし）
- `.outliner-subtext font-size`: `12px` → `0.75em`

**styles.css コンテンツ領域の相対化**:
- `.editor pre font-size`: `14px` → `0.875em`
- `.source-editor font-size`: `14px` → `0.875em`

**ビルドスクリプト修正（既存バグ修正）**:
- `build-standalone.js`: `'16px'` → `'14'`（数値のみ）+ インラインCSS `16px` → `14px`
- `build-standalone-outliner.js`: `'16px'` → `'14'` + outliner.css 置換追加
- `build-standalone-notes.js`: 同上

### テーマ影響

全7テーマ（github, sepia, night, dark, minimal, things, perplexity）で確認済み。
- outliner.css: テーマブロックに `--outliner-font-size` オーバーライドなし → 自動反映
- styles.css: 全テーマが `var(--font-size)` ベース → 自動反映
- Things テーマ: `calc(var(--font-size) - 1px)` → 14px時は13pxに自動計算
