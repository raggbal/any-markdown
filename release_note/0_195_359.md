# v0.195.359 リリースノート

## 変更内容: キーバインディング変更 + ツールチップにショートカット表示

### 課題
- 「テキストエディタで開く」ボタンのショートカットが `Cmd+.` / `Ctrl+.` だった
- 「ソースモード切替」ボタンのショートカットが `Cmd+/` / `Ctrl+/` だった
- ツールバーボタンにホバーしてもショートカットキーが表示されず、ユーザーがショートカットを知る手段がなかった

### 対策

#### キーバインディング変更 (package.json)
| コマンド | 変更前 | 変更後 |
|---|---|---|
| toggleSourceMode | `Cmd+/` / `Ctrl+/` | `Cmd+.` / `Ctrl+.` |
| openAsText | `Cmd+.` / `Ctrl+.` | `Cmd+,` / `Ctrl+,` |

#### ツールチップにショートカット情報追加 (webviewContent.ts)
- 「テキストエディタで開く」ボタン: `title` に `(Cmd+,)` / `(Ctrl+,)` を追加
- 「ソースモード切替」ボタン: `title` に `(Cmd+.)` / `(Ctrl+.)` を追加
- `process.platform` でMac/その他を判定し、適切な修飾キー名を表示

### 影響範囲
- `package.json`: keybindings セクション
- `src/webviewContent.ts`: ツールバーボタンの `title` 属性

### 注意事項
- `Cmd+,` はVSCodeのSettings(環境設定)のデフォルトショートカットだが、`when` 条件 `activeCustomEditorId == 'any-markdown.editor'` でスコープが限定されるため、通常のエディタでは元の動作が維持される
