# v0.195.537 リリースノート

## Copy Path ボタンの追加

### 課題
Markdown editorで編集中のファイルパスをクリップボードにコピーする手段がなかった。他のアプリに貼り付けたり、ターミナルで使用したりする際に不便だった。

### 対策

#### 設計
- **Standalone editor**: ツールバー右端（ソースモード切替ボタンの右）に `copyPath` ボタンを追加
- **Sidepanel editor**: ヘッダーの「Open in new tab」ボタンと「Expand」ボタンの間に既存の copy-path ボタンを移動（以前は Open in new tab の左にあった）
- HostBridge パターンに従い、webview → host へメッセージ送信 → ホスト側でクリップボードにコピーする方式

#### 実装

**VSCode拡張側:**
- `host-bridge.ts`: `copyFilePath()` メソッドをインターフェースに追加
- `vscode-host-bridge.js`: `{ type: 'copyFilePath' }` メッセージを送信
- `test-host-bridge.js`: テスト用実装を追加
- `editorProvider.ts`: `copyFilePath` メッセージを受信し `vscode.env.clipboard.writeText(document.uri.fsPath)` でコピー
- `editor-body-html.js`: standalone toolbar に `<button data-action="copyPath">` を追加、sidepanel の copy-path ボタン位置を変更
- `editor.js`: `case 'copyPath'` ハンドラ追加、SidePanelHostBridge に `copyFilePath()` スタブ追加
- `editor-utils.js`: copyPath 用 SVG アイコン追加

**i18n:**
- 全7言語（en/ja/ko/es/fr/zh-cn/zh-tw）に `copyPath` キーを追加

**Electron側:**
- `preload.ts/js`: `copyFilePath` IPC 送信を追加
- `main.ts/js`: `copy-file-path` IPC ハンドラ追加（`clipboard.writeText(filePath)`）
- `html-generator.js`: standalone toolbar に copyPath ボタン追加
- `html-generator.ts`: sidepanel の copy-path ボタン位置を変更
