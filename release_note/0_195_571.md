# v0.195.571 リリースノート

## セル内改行を含むMarkdownテーブルのペースト対応

### 課題

Markdownテーブルをプレーンテキストとしてペーストする際、セル内に生の改行（`\n`）が含まれていると、テーブルとして認識されず正しく貼り付けられなかった。

例: Notionなどからコピーした以下のようなテーブル:
```
| データセット | 前処理 & 特徴量エンジニアリング
 | 特徴量 | 学習 |
| --- | --- | --- | --- |
| バッチデータ
S3 | バッチ加工
- Processing Job | データ |
```

### 原因

`markdownToHtmlFragment()` はMarkdownを行単位で処理する。テーブル行の判定条件は「`|` で始まり `|` で終わる」こと。セル内に生の改行があると1行が複数行に分割され、各行がこの条件を満たさなくなりテーブルとして認識されない。

既存のセル内改行→`<br>`変換パスは以下の2つだったが、プレーンテキストペーストには対応していなかった:
1. テーブルセル内でのペースト（paste handler内のtableCellElement分岐）
2. 外部HTMLペースト（Turndownの`tableCellEscapePipe`ルール）

### 対策

#### 設計

ペーストハンドラのプレーンテキスト処理パスに前処理関数 `normalizeMultiLineTableCells()` を追加。`markdownToHtmlFragment()` に渡す前に、折れたテーブル行を `<br>` で結合して1行に正規化する。

#### アルゴリズム

1. テキストを行分割
2. `|` で始まるが `|` で終わらない行を「折れたテーブル行」として検出
3. 後続行を `<br>` で結合しながら、`|` で終わる行に到達するまで継続（最大50行）
4. 連続する `<br>` を1つに圧縮
5. `|` で終わる行が見つからなければ結合を中止し元の行をそのまま出力

#### 実装

- `editor-utils.js`: `normalizeMultiLineTableCells()` 関数を追加（純粋関数）
- `editor.js`: ペーストハンドラのプレーンテキスト分岐で `normalizeMultiLineTableCells()` を呼び出し
- 既存の `parseInline()` が `<br>` リテラルを実際の `<br>` 要素に復元し、`processCellContent()` が `<br>` 要素を `<br>` 文字列としてMarkdown出力するため、ラウンドトリップが保証される

### テスト

`test/specs/paste-multiline-table.spec.ts` に6件のテストを追加:
1. ヘッダー行にセル内改行があるテーブル
2. データ行に複数改行があるテーブル
3. SageMaker風の複雑なテーブル
4. 通常テーブル（デグレなし確認）
5. セル内に空行を含むテーブル
6. Markdownラウンドトリップの検証
