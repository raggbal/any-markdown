# v0.195.416 - サイドパネル画像パス破損修正 + Cmd+Vペーストリレー完成

## 課題1: 画像パスにvscode-resource URIプレフィックスが残る

### 問題

サイドパネルでD&Dした画像をメインエディタで開き直すと、画像パスに `https://file%2B.vscode-resource.vscode-cdn.net/...` というプレフィックスが付いた状態で保存される。

例: `![image](https://file%2B.vscode-resource.vscode-cdn.net/Users/raggbal/test3/image.png)` と保存されるべきところが、正しくは `![image](test3/image.png)` であるべき。

### 原因

`contenteditable` 内の `<img>` 要素は2つの属性を持つ:
- `src`: 表示用URI（vscode-resource URI）
- `data-markdown-path`: Markdown保存用のオリジナルパス

HTML→Markdown変換時に `data-markdown-path` が利用できない場合（ブラウザのundo/redoでdata属性が失われる等）、`src` がフォールバックとして使われ、vscode-resource URIがそのままMarkdownに出力されていた。

### 対策

**設計**: `cleanImageSrc()` セーフティネット関数を追加し、HTML→Markdown変換の全3箇所に適用。

**実装**:
- `cleanImageSrc(src)`: vscode-resource URIプレフィックス（`file+` / `file%2B` の両バリエーション）を除去、data: URLは空文字に変換
- 適用箇所:
  1. `mdProcessNode()` の `<img>` 処理 (line 5521)
  2. `collectCharStyles()` の `<img>` 処理 (line 5646)
  3. `mdProcessTable()` の `<img>` 処理 (line 6012)
- すべて `cleanImageSrc(node.dataset.markdownPath || node.getAttribute('src') || '')` の形で、markdownPath優先 + srcフォールバック + cleanImageSrcで保護の3段構え

## 課題2: サイドパネルでCmd+Vペーストが効かない

### 問題

サイドパネルiframe内でCmd+Vを押しても何も反応しない。テキストも画像も貼り付けできない。

### 原因

サイドパネルはblob: URLで読み込まれたiframe。blob: オリジンでは `navigator.clipboard.read()` が `NotAllowedError: Read permission denied` で拒否される。

v0.195.415で iframe内のkeydownハンドラに `requestPaste` メッセージ送信は追加済みだったが、**親webviewのメッセージリスナーに `requestPaste` ハンドラが未実装**だった。iframe→parent→クリップボード読取→iframe へのリレーチェーンが途切れていた。

### 対策

**設計**: iframe→parent→iframe のクリップボードリレー方式。

1. iframe内Cmd+V → `parent.postMessage({source:'sidePanel', type:'requestPaste'})` （v0.195.415で実装済み）
2. 親webviewのiframeメッセージリスナーで `requestPaste` を受信
3. 親webview（vscode-resource オリジン）で `navigator.clipboard.read()` を実行
4. 画像の場合: FileReader → data URL → `host.saveImageAndInsert()` → `insertImageHtml` リレー
5. テキストの場合: `navigator.clipboard.readText()` → `pasteText` メッセージでiframeに送信
6. iframe内の既存 `pasteText` ハンドラがテキストを挿入

**実装**: 親webviewの `window.addEventListener('message')` 内（sidePanel source ハンドラ）に `requestPaste` ケースを追加。画像判定→テキストフォールバックの2段階処理。
