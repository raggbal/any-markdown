# v0.195.345 リリースノート

## Perplexityテーマのコードブロック シンタックスハイライト色修正

### 課題
Perplexityテーマでコードブロック（javascript等）のシンタックスハイライトの色が視認できない。`function`や`const`などのキーワードが、本来は赤色(`#d73a49`)で表示されるべきところ、ほぼ黒に見える。

### 原因
Perplexityテーマ専用のhljs色定義が存在し、デフォルトの色を上書きしていた。

```css
[data-theme="perplexity"] .hljs-keyword { color: #1A6872; }
```

この`#1A6872`（暗いティール）は、Perplexityテーマのベーステキスト色`#13343B`（暗いティール）と非常に近く、キーワードが黒文字と見分けがつかなかった。

他のライト系テーマ（github, sepia, minimal）はデフォルトの色（keyword=赤`#d73a49`等）をそのまま使用しており、dark テーマのみが背景が暗いため独自の明るい色を定義している。

### 対策

#### 設計
Perplexityはライト系テーマ（背景`#FBFAF4`）であり、デフォルトのhljs色（GitHub準拠）で十分視認できる。Perplexityテーマ独自のhljs色定義を全て削除し、他のライト系テーマと同様にデフォルト色を使用する。

#### 実装
`src/webview/styles.css` から以下の13行を削除:
- `[data-theme="perplexity"] .hljs-keyword` 〜 `[data-theme="perplexity"] .hljs-property`

これにより、デフォルトの`.hljs-keyword { color: #d73a49; }`等が適用される。
