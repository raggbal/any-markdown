# v0.195.503 — Outliner ページタイトル機能

## 課題

Outlinerで開いた `.out` ファイルにタイトル（ドキュメント名）を付ける手段がなかった。ノードのテキストで代用するしかなく、ドキュメントの識別性が低かった。

## 対策

### 要件

- VSCode設定 `any-markdown.outlinerPageTitle`（boolean, default: false）を追加
- `true` の場合、検索バー・パンくずリストより上にタイトル入力欄を表示
- `.out` ファイル（JSON）のトップレベルに `title` フィールドを追加
- 設定を `false` に戻しても、JSON内の `title` は削除せず保持

### 設計

#### データモデル（outliner-model.js）
- `Model` コンストラクタで `data.title` を読み込み（未定義なら空文字）
- `serialize()` で `title` が空でなければJSONに含める
- 設定のon/offに関わらず、値がある限りJSONに残り続ける

#### 設定フロー（outlinerProvider.ts → outlinerWebviewContent.ts）
- `OutlinerConfig` インタフェースに `outlinerPageTitle` を追加
- `updateWebview()` で設定値を読み取りHTMLオプションに渡す
- HTML生成時に `style="display:none"` で初期非表示 / 設定trueで表示
- `onDidChangeConfiguration` に `outlinerPageTitle` を追加（変更時にwebview再生成）

#### UI（outliner.js + outliner.css）
- `.outliner-page-title` コンテナ + `<input>` 要素をbreadcrumbの上に配置
- `input` イベントで `model.title` を更新 → `scheduleSyncToHost()`（1秒デバウンス）で保存
- IME対応: `compositionstart`/`compositionend` でガード
- Enter キーでツリーの最初のノードにフォーカス移動
- 外部変更（`updateData`）時はタイトル入力中でなければ値を復元
