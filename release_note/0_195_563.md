# v0.195.563 リリースノート

## サイドパネルの toolbarMode 設定が Outliner/Notes で無視されるバグ修正

### 課題

VSCode設定 `fractal.toolbarMode` を `simple` に設定しても、Outliner editor や Notes editor からサイドパネルの Markdown editor を開くと、ツールバーが常に `full` モードで表示される。

- Outliner → サイドパネル Markdown editor: 常に full
- Notes → Outliner → サイドパネル Markdown editor: 常に full
- 通常の Markdown editor (.md 直接): 正常に simple が適用される

### 原因

`toolbarMode` 設定の読み取り・HTML注入が Markdown editor (`webviewContent.ts` / `editorProvider.ts`) にしか実装されておらず、Outliner/Notes の HTML 生成パスに欠落していた。

CSS は `[data-toolbar-mode="simple"] .toolbar-inner { display: none }` で `<html>` タグの `data-toolbar-mode` 属性を参照して制御しているが、Outliner/Notes の `<html>` タグにはこの属性が存在しなかった。

| ファイル | 問題 |
|---|---|
| `outlinerWebviewContent.ts` | `OutlinerConfig` に `toolbarMode` なし、`<html>` に `data-toolbar-mode` なし |
| `notesWebviewContent.ts` | `NotesConfig` に `toolbarMode` なし、`<html>` に `data-toolbar-mode` なし |
| `outlinerProvider.ts` | config で `toolbarMode` を読み取っていない |
| `notesEditorProvider.ts` | config で `toolbarMode` を読み取っていない |

### 対策

#### 設計

既存の Markdown editor (`webviewContent.ts`) と同じパターンを Outliner/Notes にも適用:
1. config interface に `toolbarMode` を追加
2. provider で VSCode 設定から `toolbarMode` を読み取り
3. `<html>` タグに `data-toolbar-mode` 属性を注入

#### 実装

| ファイル | 変更内容 |
|---|---|
| `src/outlinerWebviewContent.ts` | `OutlinerConfig` に `toolbarMode?: string` 追加、`<html>` タグに `data-toolbar-mode="${config.toolbarMode \|\| 'full'}"` 追加 |
| `src/notesWebviewContent.ts` | `NotesConfig` に `toolbarMode?: string` 追加、`<html>` タグに `data-toolbar-mode="${config.toolbarMode \|\| 'full'}"` 追加 |
| `src/outlinerProvider.ts` | `config.get<string>('toolbarMode', 'full')` で設定読み取り |
| `src/notesEditorProvider.ts` | `config.get<string>('toolbarMode', 'full')` で設定読み取り |
