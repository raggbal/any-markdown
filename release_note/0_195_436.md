# v0.195.436 - サイドパネル画像パス修正 + IMAGE_DIR設定ボタン追加

## 課題

サイドパネルエディタで画像を挿入（ペースト/D&D/ツールバー）すると、画像の保存先ディレクトリとMarkdownパスが**メインエディタのmdファイル基準**で解決されていた。サイドパネルのmdファイルから見ると画像パスが間違っており、画像が正しく表示されない。

また、サイドパネルにはIMAGE_DIR設定UIがなく、サイドパネルファイル個別の画像ディレクトリを設定できなかった。

## 原因

`SidePanelHostBridge`の画像メソッド3つ（`requestInsertImage`, `saveImageAndInsert`, `readAndInsertImage`）が、サイドパネルのファイルパスを渡さずにメインhostにそのまま委譲していた。

```javascript
// 修正前: サイドパネルのfilePathが一切使われていない
requestInsertImage() {
    this._mainHost.requestInsertImage();  // メインドキュメント基準で処理される
}
```

`editorProvider.ts`の画像ハンドラも常にメインドキュメント（`document: vscode.TextDocument`）を使って画像ディレクトリ解決・パス計算を行っていた。`sidePanelDocument`/`sidePanelWatchedPath`は存在したが、ファイル監視と保存にしか使われていなかった。

## 対策

### 設計

メッセージチェーン全体にoptional `sidePanelFilePath`パラメータを追加し、サイドパネルからのリクエストを識別できるようにした。

```
SidePanelHostBridge.saveImageAndInsert(dataUrl, fileName)
  → this._mainHost.saveImageAndInsert(dataUrl, fileName, this.filePath)  // SP path追加
  → vscode-host-bridge.js: { type: 'saveImageAndInsert', ..., sidePanelFilePath }
  → editorProvider.ts: sidePanelFilePath有→SPファイル基準で解決 / 無→メイン基準
```

### 実装（7ファイル変更）

**1. インターフェース層（3ファイル）**
- `host-bridge.ts`: `requestInsertImage`, `requestSetImageDir`, `saveImageAndInsert`, `readAndInsertImage`にoptional `sidePanelFilePath`パラメータ追加
- `vscode-host-bridge.js`: 上記4メソッドで`sidePanelFilePath`をメッセージに含めて送信
- `test-host-bridge.js`: 同上（CLAUDE.md Section 26 rule 2準拠）

**2. SidePanelHostBridge（editor.js）**
- 画像メソッド3つで`this.filePath`を渡すように修正
- `requestSetImageDir()`を空実装から`this._mainHost.requestSetImageDir(this.filePath)`に変更

**3. 画像ハンドラリファクタ（editorProvider.ts）**
- `resolveImageDocumentUri()`: `sidePanelFilePath`有無でURI解決
- `resolveImageDocumentContent()`: `sidePanelDocument`バッファ優先、なければディスク読み取り
- `handleImageInsert`, `handleSaveImage`, `handleReadAndInsertImage`: `document: TextDocument`から`documentUri: Uri + documentContent: string`方式に変更
- `setImageDir`ハンドラもSP対応

**4. サイドパネルUI（editor-body-html.js + editor.js）**
- サイドパネルヘッダーに`imageDir`ボタン追加（undo/redo/sourceと並列）
- クリックハンドラで`sidePanelHostBridge.requestSetImageDir()`呼び出し
- `toolbarTitleMap`に`imageDir`エントリ追加（i18n対応済み）

**5. Electron対応（preload.ts）**
- 画像メソッド4つに`sidePanelFilePath`パラメータ追加

### 後方互換性

- メインエディタからの呼び出しでは`sidePanelFilePath`は`undefined`のため既存動作に変更なし
- メッセージ型は同一（optionalプロパティ追加のみ）
