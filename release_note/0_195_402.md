# v0.195.402 - サイドパネルの外部変更上書き問題の修正

## 課題

サイドパネル（iframe方式）で開いたMarkdownファイルが、外部エディタ（Claude Code、別のVSCodeウィンドウ等）から変更された場合、サイドパネルが古い内容で上書きしてしまう。ユーザーがサイドパネル内で一切編集していなくても発生する。

## 原因

### アーキテクチャの差異

メインエディタとサイドパネルで保存先のレイヤーが異なっていた。

**メインエディタ（問題なし）:**
```
syncContent(md) → VSCode TextDocument (メモリ) → ディスク
                      ↕ 別レイヤー
FileSystemWatcher → ディスク → TextDocument更新 → webview更新
```

**サイドパネル（問題あり）:**
```
syncContent(md) → fs.writeFile() → ディスク  ← 直接書き込み！
                                      ↕ 同じレイヤー
FileSystemWatcher → ディスク → isSidePanelSaving? → スキップ！
```

### 具体的な発生メカニズム

1. サイドパネルが開く → `init()` → `renderFromMarkdown()` → `setupInteractiveElements()`
2. `setupInteractiveElements()` がコードブロックヘッダー等を `contenteditable` 内に DOM 挿入
3. ブラウザが `input` イベントを発火 → `markAsEdited()` → `hasUserEdited = true`
4. `debouncedSync()` (1000ms) がスケジュールされる
5. 1000ms後、古い内容で `host.syncContent()` → `saveSidePanelFile()` → `fs.writeFile()` でディスクに直接保存
6. `isSidePanelSaving = true` (300ms) により、同時期の FileSystemWatcher イベントがスキップされる
7. 結果: 外部変更がサイドパネルの古いデータで上書きされる

メインエディタでは VSCode TextDocument がバッファ層として機能するため同じ問題が顕在化しない。

## 対策

### 設計

サイドパネルもメインエディタと同じアーキテクチャ（TextDocument バッファ経由）に統一。

```
【修正後のサイドパネル】
syncContent(md) → VSCode TextDocument (メモリ) → save() → ディスク
                      ↕ 別レイヤー
FileSystemWatcher → TextDocument同期 → onDidChangeTextDocument → iframe中継
```

### 実装

#### 変更ファイル

**src/editorProvider.ts** のみ:

1. **変数宣言の変更:**
   - `isSidePanelSaving` (boolean) → `isApplyingSidePanelEdit` (boolean) — メインエディタの `isApplyingOwnEdit` と同じパターン
   - `sidePanelDocument` (TextDocument) 追加 — サイドパネルファイルのインメモリバッファ
   - `sidePanelDocChangeSubscription` (Disposable) 追加 — TextDocument変更監視

2. **`setupSidePanelFileWatcher()` の変更:**
   - `vscode.workspace.openTextDocument(fileUri)` で TextDocument を取得（タブは開かない）
   - FileSystemWatcher: 外部変更検知 → TextDocument を `WorkspaceEdit` で同期（メインエディタと同パターン）
   - `onDidChangeTextDocument`: TextDocument変更 → iframe に `sidePanelMessage` で中継

3. **`saveSidePanelFile` ハンドラの変更:**
   - `fs.writeFile()` → `WorkspaceEdit` + `document.save()` に変更
   - 保存前に内容比較（`normalize` で CRLF 正規化）、同一なら skip
   - TextDocument が未取得の場合は fallback として直接書き込み

4. **`setupSidePanelFileWatcher` が async に:**
   - `openTextDocument` が非同期のため
   - 呼び出し元2箇所（`openLink`, `sidePanelOpenLink`）に `await` 追加
