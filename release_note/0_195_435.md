# v0.195.435 - コードブロックCopyボタンの改行消失修正

## 課題

コードブロックのCopyボタンをクリックしてコピーすると、改行コードが消えた状態でクリップボードにコピーされる。

## 原因

Copyボタンのクリックハンドラで `code.textContent` を使用していた。

`contenteditable` 内のコードブロックでは、ユーザーがEnterで入力した改行は `<br>` 要素として表現される（CLAUDE.md Section 21参照）。`textContent` は `<br>` 要素を無視してテキストノードのみを結合するため、`<br>` で表現された改行が消失していた。

## 対策

### 設計

既存の `getCodePlainText(code)` 関数を使用する。この関数はコードブロック内のテキストノードと `<br>` 要素の両方を走査し、`<br>` を `\n` に変換してプレーンテキストを生成する。シンタックスハイライトの `<span>` も正しくスキップする。

### 実装

`src/webview/editor.js` L3257:
- Before: `const text = code.textContent || '';`
- After: `const text = getCodePlainText(code);`

1行の変更のみ。`getCodePlainText` は既にedit/displayモード切替で使用されている既存関数であり、新規コードの追加は不要。
