# v0.195.575 リリースノート

## .md ファイルの Outliner D&D 取り込み機能

### 課題

Notion / Obsidian 等から出力した `.md` ファイルを Outliner エディタに取り込む手段がなかった。
ユーザーはファイルの内容を手動でコピー&ペーストし、ページノードを手動で作成する必要があった。

### 対策

#### 設計

OS ファイルマネージャー（Finder / Explorer）や VSCode Explorer から `.md` ファイルを Outliner のツリーにドラッグ&ドロップして、ページノードとして一括取り込みする機能を追加。

アーキテクチャ:
1. **Webview側** (outliner.js): 既存ノードD&Dのdragover/dropハンドラを拡張し、外部ファイルドロップを検出。`hasExternalFiles(e)` で `dataTransfer.types` に `'Files'` があるか判定。既存のノードD&D (`dragState`) / 画像D&D (`imageDragState`) とは完全分離。
2. **Host側** (markdown-import.ts): 新規共通モジュール。ファイル読み込み→H1抽出→変換処理→画像コピー→ページ保存を一括実行。
3. **メッセージング**: `importMdFiles` → Host処理 → `importMdFilesResult` → ノード作成

#### 実装

**新規ファイル:**
- `src/shared/markdown-import.ts` — 変換処理・画像コピーの共通ロジック
  - `normalizeMarkdownPlainText()`: editor-utils.js の `normalizeMultiLineTableCells()` と同等ロジック（Node.js環境用）
  - `normalizeMarkdownFromHtml()`: 将来のHTML D&D対応用スタブ
  - `extractH1Title()`: 最初のH1テキスト抽出（なければ "Untitled"）
  - `processImages()`: 画像パス解析→コピー（`image_{timestamp}_{random}.{ext}` にリネーム）→パス書き換え
  - `importMdFile()` / `importMdFiles()`: 統合関数

**変更ファイル:**
- `src/webview/outliner.js`:
  - `hasExternalFiles(e)`: 外部ファイルドラッグ判定
  - `handleExternalFileDrop(e, targetEl)`: .mdフィルタ、ファイル名ソート、位置計算、host呼び出し
  - 各ノード/treeEl の dragover/drop ハンドラ拡張（外部ファイル時もインジケーター表示）
  - `importMdFilesResult` メッセージ受信処理（ノード作成・挿入）
- `src/outlinerProvider.ts`: `importMdFiles` case 追加
- `src/shared/notes-message-handler.ts`: `importMdFiles` case 追加
- `src/shared/outliner-host-bridge.js`: `importMdFiles()` メソッド追加
- `src/shared/notes-host-bridge.js`: `importMdFiles()` メソッド追加

**ドロップ位置:**
- ノード上部25%: before（兄弟として前に挿入）
- ノード下部25%: after（兄弟として後に挿入）
- ノード中央50%: child（子として挿入、自動展開）
- ツリー空エリア: ルート末尾

**複数ファイル対応:**
- ファイル名でソートし順序通りに挿入
- 2番目以降は前のノードの直後に兄弟として挿入

**画像処理:**
- `![alt](path)` パターンを検出
- URL（http/https）はスキップ
- 元ファイルからの相対パスで解決→存在すればコピー→ページからの相対パスに書き換え
- 保存先: Standalone = `{pageDir}/images/`, Notes = `{outlineId}/images/`
