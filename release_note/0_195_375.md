# v0.195.375: コードブロック言語変更で作成したmermaid/mathブロックがクリックで編集モードにならない問題を修正

## 課題

コードブロックを作成した後、言語セレクターから「mermaid」または「math」を選択してspecial wrapperに変換した場合、そのブロックをクリックしても編集モードに入らない。

一方、`` ```mermaid `` や `` ```math `` で直接作成したブロックは正常にクリックで編集モードに入る。

## 原因

`convertToSpecialBlock()` 関数（言語セレクターからmermaid/mathを選択した際に呼ばれる）に、イベントハンドラの登録が漏れていた。

special wrapperの動作に必要な3つのイベントハンドラ:

| ハンドラ | 役割 |
|---|---|
| `click` | display → edit モード切替 |
| `input` | 入力時のライブ再レンダリング（500msデバウンス） |
| `focusout` | edit → display モード切替 |

これらは以下の3箇所では登録されていたが、`convertToSpecialBlock()` では未登録だった:

1. `setupMermaidDiagrams()` / `setupMathBlocks()` — 初期レンダリング時（L2597, L2710）
2. `` ```mermaid `` / `` ```math `` パターン変換時（L4507, L4583）

さらに `convertToSpecialBlock()` は `wrapper.dataset[type + 'Setup'] = 'true'` を設定するため、後から `setupMermaidDiagrams()` / `setupMathBlocks()` が呼ばれても「セットアップ済み」としてスキップされ、ハンドラが永久に登録されない状態だった。

CLAUDE.md Section 3（修正対象の概念を grep で網羅的に洗い出す）の教訓に該当するケース。

## 対策

### 設計

`convertToSpecialBlock()` 内で、他の作成パスと同じ3つのイベントハンドラを登録する。

### 実装

`src/webview/editor.js` の `convertToSpecialBlock()` 関数（L2999付近）に以下を追加:

```javascript
// click handler — 編集モード進入
wrapper.addEventListener('click', function(e) {
    if (wrapper.getAttribute('data-mode') !== 'edit') {
        wrapper.setAttribute('data-mode', 'edit');
        newCode.focus();
        setCursorToEnd(newCode);
    }
});

// input handler — ライブ再レンダリング
var renderTimeout = null;
newPre.addEventListener('input', function() {
    if (renderTimeout) clearTimeout(renderTimeout);
    renderTimeout = setTimeout(function() {
        renderFn(wrapper);
    }, 500);
});

// focusout handler — 表示モード復帰
newCode.addEventListener('focusout', function(e) {
    setTimeout(function() {
        if (!wrapper.contains(document.activeElement)) {
            if (wrapper.getAttribute('data-mode') === 'edit') {
                wrapper.setAttribute('data-mode', 'display');
                renderFn(wrapper);
                syncMarkdown();
            }
        }
    }, 100);
});
```

### テスト

`test/specs/codeblock-lang-to-special-wrapper.spec.ts` を新規作成（3テスト）:
1. mermaid wrapper converted from code block enters edit mode on click
2. math wrapper converted from code block enters edit mode on click
3. mermaid wrapper converted from code block exits edit mode on focusout

既存テスト38件もパス確認済み。
