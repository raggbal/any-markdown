# v0.195.432 - Redo完全修正 + サイドパネルヘッダーバー + デッドコード削除

## 課題1: Cmd+Shift+Z (Redo) がVSCode内で動作しない

### 問題

Cmd+Shift+Z (redo) がメインエディタ・サイドパネルの両方で動作しない。DevToolsを開いているときだけ動作する。

### 原因

VSCodeのElectronレベルがCmd+Shift+Zを横取りし、DOM keydownイベントがwebviewに到達しない。Cmd+ZはDOMに到達するが、Cmd+Shift+ZはVSCodeが先に処理する。

### 対策

**設計**: 2経路並行方式を採用。

| 経路 | 対象 | フロー |
|---|---|---|
| DOM capture | DevTools開時 / 一部環境 | keydown capture → EditorInstance._undo/_redo |
| VSCode keybinding | 通常時 | keybinding → extension.ts → postMessage → editor.js message handler |

**実装**:
- `package.json`: redo keybindingの`when`句を`activeCustomEditorId == 'any-markdown.editor'`から`!editorTextFocus`に変更（VSCodeがElectronレベルで横取りする場合でもkeybinding発火）
- `extension.ts`: `sendRedo()`がfalse返却時にbuilt-in `redo`にフォールバック
- `editorProvider.ts`: `sendUndo()`/`sendRedo()`をboolean返却に変更
- `editor.js`: `_lastUndoRedoTime`による100ms dedup guard（DOM handler + VSCode keybinding両方発火時の二重実行防止）

## 課題2: サイドパネルでのRedo動作不良

### 問題

VSCode keybinding経由のperformRedoメッセージが、サイドパネルインスタンスではなくメインインスタンスに届く。

### 原因

VSCodeがCmd+Shift+Zを横取りする際、`document.activeElement`が`document.body`になる。`getActiveInstance()`は`body.contains(body) === true`で常にメインインスタンスにマッチし、サイドパネルを見落とす。

### 対策

- `EditorInstance._lastKnownActive`: editor/sourceEditorのfocusイベントで最後にフォーカスされたインスタンスを記録
- `getActiveInstance()`: `document.activeElement === body`の場合、`_lastKnownActive`を優先返却
- `destroy()`: `_lastKnownActive`が破棄されたインスタンスを指す場合のクリーンアップ追加
- `performUndo`/`performRedo`/`toggleSourceMode`メッセージハンドラ: `EditorInstance.getActiveInstance()`経由でアクティブインスタンスのメソッドを呼び出し

## 課題3: 入力直後のUndo→Redoが動作しない（タイミング依存）

### 問題

文字入力後すぐにCmd+Z → Cmd+Shift+Zすると、redoが正しく動作しない。2秒待ってからundo→redoすると正常。

### 原因

`markdown`変数は`debouncedSync()` (1000ms debounce)でのみ更新。`undo()`内の`capture()`が呼ばれた時点でdebounce未発火の場合、`markdown`は入力前の古い値のまま。redoスタックにundoと同じ状態が積まれ、redoしても変化なし。

### 対策

`undo()`内で`capture()`前に`markdown = htmlToMarkdown()`を実行。現在のDOM状態から直接Markdownを生成し、debounceに依存しない正確なキャプチャを保証。

## 課題4: サイドパネルにツールバーが表示されない

### 問題

サイドパネルのEditorInstanceコンテナに空のtoolbar divのみ生成されており、ボタンが一切表示されなかった。

### 対策

- `createSidePanelContainer()`のtoolbar HTMLをメインツールバーと同じボタン構成に置換（`openOutline`/`openInTextEditor`除外）
- `toolbarTitleMap`でi18nタイトルを動的設定
- `undoManager.updateButtons()`を`document.querySelector` → `container.querySelector`に変更（インスタンス固有スコープ）

## 課題5: サイドパネルヘッダーにundo/redo/sourceボタン追加

### 対策

- `editor-body-html.js`: `.side-panel-header-actions`にundo/redo/sourceボタンを追加
- `styles.css`: `.side-panel-header-btn`スタイル追加
- `editor.js`: `setupSidePanelHeaderButtons()`でイベントハンドラ登録、`_setUndoUpdateCallback`でヘッダーボタン状態同期
- simpleモード時はtoolbar非表示でヘッダーボタンのみ表示

## 課題6: サイドパネル保存時の「Document has been closed」エラー

### 問題

サイドパネルのファイル保存・外部変更検知時にVSCodeが自動的にTextDocumentを閉じることがあり、`applyEdit`が失敗する。

### 対策

`editorProvider.ts`の3箇所（FileSystemWatcher、onDidChangeTextDocument、saveSidePanelFile）で`sidePanelDocument.isClosed`チェックを追加。closedの場合は`openTextDocument()`で再オープン。

## 課題8: ショートカットのインスタンス委譲

### 問題

document-levelのkeydownハンドラがメインインスタンスのクロージャ変数を直接参照しており、サイドパネルでショートカットが正しく動作しない。

### 対策

- `_handleGlobalShortcut`メソッドとして各インスタンスに公開
- document-levelハンドラは`EditorInstance.getActiveInstance()`でアクティブインスタンスを解決し、そのインスタンスの`_handleGlobalShortcut`を呼び出し
