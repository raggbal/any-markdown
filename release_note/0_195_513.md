# v0.195.512-513 — Notes フォルダ管理 + ファイル切替時データ消失バグ修正

## 新機能: Notesフォルダ/ツリー管理 (v0.195.512)

### 課題
左パネルのファイル一覧がフラットリストのみで、フォルダによる分類・D&Dによる並び替えができなかった。

### 設計
- 物理フォルダは作成せず、`.note` JSONファイル1つでツリー構造を管理
- `.note` のデータ構造は outliner の `.out` と類似: `rootIds[]` + `items{}` マップ
- アイテム型: `file`（.outへの参照）と `folder`（仮想フォルダ）
- ディスク上の `.out` と `.note` を自動同期（孤児追加・欠損除去）

### 実装

**データ層** (`notes-file-manager.ts`):
- `NoteStructure` インターフェース: `{ version, rootIds, items }`
- `NoteTreeFile`: `{ type: 'file', id, title }`
- `NoteTreeFolder`: `{ type: 'folder', id, title, childIds, collapsed }`
- `loadStructure()`: .noteファイル読み込み + ディスク同期
- `createFolder()`, `deleteFolder()`, `renameFolder()`, `toggleFolderCollapsed()`
- `moveItem()`: D&D移動（循環参照チェック `isDescendant()` 付き）
- `findFirstFileId()`: ツリー順DFS で最初のファイルを返す

**メッセージ層** (`notes-message-handler.ts`):
- 5つの新メッセージ: `notesCreateFolder`, `notesDeleteFolder`, `notesRenameFolder`, `notesToggleFolder`, `notesMoveItem`
- `sendFileListWithStructure()` ヘルパーで全レスポンスに `structure` を含める

**ブリッジ** (`notes-host-bridge.js`):
- `createFolder()`, `deleteFolder()`, `renameFolder()`, `toggleFolder()`, `moveItem()` 追加
- `onFileListChanged` ハンドラが `structure` も受信

**UI** (`notes-file-panel.js`):
- `renderTree()`: 再帰レンダリング（フォルダ展開/折りたたみ）
- フォルダ: シェブロン + フォルダアイコン + タイトル、クリックで展開切替
- D&D: `draggable="true"` + module-scope変数（VSCode webview互換）
- ドロップ判定: 上25%=前挿入、中央50%=フォルダ内、下25%=後挿入
- コンテキストメニュー: フォルダ（新規Outline/サブフォルダ/リネーム/削除）

**CSS** (`notes-body-html.js`):
- `.file-panel-folder`, `.file-panel-folder-header`, `.file-panel-folder-children`
- `.file-panel-drag-over`, `.file-panel-drop-line`
- ファイルアイコン、フォルダアイコン、シェブロンのSVG

## バグ修正: ファイル切替時のデータ消失 (v0.195.513)

### 課題1: stale syncData によるファイル上書き
ファイル切替後、webview側のデバウンスタイマーが旧ファイルのデータを送信し、新ファイルの `currentFilePath` に書き込まれてしまう。

### 原因1
webview側の `syncData` デバウンス（1秒）とホスト側の `currentFilePath` 変更が非同期のため、タイミング次第でデータが別ファイルに書き込まれる。

### 対策1: fileChangeId カウンター
- `NotesFileManager` に `fileChangeId` を追加、`openFile()` ごとにインクリメント
- `updateData` メッセージに `fileChangeId` を含める
- `notes-host-bridge.js` が `updateData` から `fileChangeId` を記憶し `syncData` に付与
- `syncData` ハンドラで `fileChangeId` が一致しない場合は無視

### 課題2: 改行前のテキスト入力がファイル切替で消失
入力後にEnterを押さず即座に別ファイルを選択すると、webview側のデバウンスが未発火のため入力内容が保存されない。

### 原因2
ホスト側の `flushSave()` はホスト側タイマーのみフラッシュし、webview側のデバウンスタイマーには効かない。

### 対策2: ファイル切替前の即時同期
- `Outliner` 公開APIに `flushSync()` を追加（内部の `syncToHostImmediate()` を呼ぶ）
- `notes-host-bridge.js` の `openFile()` が `flushOutlinerSync()` → `Outliner.flushSync()` を呼び、未保存データ（`searchFocusMode` + `pageDir` 含む）を即座に送信してからファイル切替メッセージを送る
