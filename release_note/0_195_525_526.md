# v0.195.525〜526 リリースノート

## 概要
致命的データ消失バグの修正、フォーカスモード検索のスコープ修正、Undo/Redoボタン追加。

---

## 1. 【致命的】ファイル切替時のUndo/Redoによるデータ上書きバグ修正

### 課題
Notesモードでファイル切替後にCmd+Z(Undo)を押すと、前のファイルのデータで現在のファイルが上書きされる。タイトルまで上書きされ、データ消失が発生する致命的バグ。

### 原因
`undoStack` / `redoStack` がファイル切替時にクリアされていなかった。ファイルAを編集→ファイルBに切替→Cmd+Zで、ファイルAのスナップショットがファイルBのmodelに適用され、`syncToHostImmediate()` でホストに送信されていた。

### 対策（設計）
`updateData`受信時にundo/redoスタックを両方クリアし、新データの初期スナップショットを保存する。

### 対策（実装）
`outliner.js` の `updateData` メッセージハンドラ内で:
```javascript
undoStack.length = 0;
redoStack.length = 0;
// ... model再構築後
saveSnapshot();
```

## 2. フォーカスモード検索のスコープ制限修正

### 課題
Scope-in状態でフォーカスモード検索を実行すると、スコープ外のノードもマッチ結果に表示される。

### 原因
`renderFocusNodes()` が `Object.keys(model.nodes)`（全ノード）を候補にしていた。`currentScope` を考慮していなかった。

### 対策（実装）
候補ノードIDリストを `currentScope` で制限:
```javascript
var candidateIds;
if (currentScope.type === 'subtree' && currentScope.rootId) {
    candidateIds = [currentScope.rootId].concat(model.getDescendantIds(currentScope.rootId));
} else {
    candidateIds = Object.keys(model.nodes);
}
```

## 3. Undo/Redoボタン追加

### 課題
Outliner EditorにUndo/Redoのボタンがなく、キーボードショートカットのみだった。

### 対策（設計）
検索バーの右端（メニューボタンの左）にUndo/Redoボタンを配置。スタック状態に応じてdisabled制御。

### 対策（実装）
- **HTML**: `outlinerWebviewContent.ts` / `notesWebviewContent.ts` の検索バーにボタン2個追加
- **CSS**: `outliner.css` にボタンスタイル追加（disabled時の半透明化含む）
- **JS**: `outliner.js` に以下を追加:
  - `setupUndoRedoButtons()` でイベントリスナー登録（`mousedown` で `preventDefault` してフォーカス維持）
  - `updateUndoRedoButtons()` でスタック状態に応じたdisabled制御
  - `saveSnapshot()`, `undo()`, `redo()`, `updateData` ハンドラの各所から `updateUndoRedoButtons()` を呼び出し
