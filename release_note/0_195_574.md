# v0.195.574 リリースノート

## HTMLペースト時のTurndownインラインエスケープ除去

### 課題

リッチテキストソース（Notion、ブラウザ等）からMarkdownをペーストすると、Turndownが不要なバックスラッシュエスケープを挿入する。

例: `## 1. サービス概要` → `## 1\. サービス概要・位置づけ`

### 原因

Turndownは HTML→Markdown 変換時に、生成したMarkdownが再パースされた時に意図しない構文解釈を防ぐため、テキストノード内のMarkdown特殊文字をエスケープする（`vendor/turndown.js` L732-745 の `escapes` 配列）。

既存のエスケープ除去（v0.195.570で追加）は**行頭のブロックレベル構文**（`\-`, `\+`, `\#`, `\>`, `数字\.`, `\~~~`）のみ対応していた。しかし以下のケースが未対応だった:

1. **行頭以外の `\.`**: `## 1\. サービス概要` — 見出しマーカー後にある `1\.` は行頭正規表現 `^(\d+)\\(\. )` にマッチしない
2. **インライン特殊文字**: `\*`, `\_`, `` \` ``, `\[`, `\]`, `\\` — テキストノード内の全出現がエスケープされる

### 対策

Turndown変換後の既存ブロックレベルエスケープ除去の直後に、インラインエスケープ除去を1行追加:

```javascript
pastedMd = pastedMd.replace(/\\([*_`\[\]\\.])/g, '$1');
```

対象文字: `*`, `_`, `` ` ``, `[`, `]`, `\`, `.`

### テスト

`html-paste-unescape-block-markers.spec.ts` に2件追加（計9件）:
- `## 1\. サービス概要` → `## 1. サービス概要`（行頭以外の `\.` 除去）
- `\*`, `\_`, `` \` ``, `\[`, `\]`, `\\` のインラインエスケープ除去
