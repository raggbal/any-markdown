# v0.195.459 - ペースト時の先頭インデント調整

## 課題

コピーした範囲の先頭行が最小インデントでない場合、ペースト時にインデント構造が壊れる。

### 再現例

```
- a
  - b      ← level 1
    - c    ← level 2
- d        ← level 0 (最小)
```

b, c, d をコピーして別の場所にペーストすると、b(level 1) に対する親ノード（level 0）が存在せず、ツリー構造が壊れる。

### 期待する結果

最小インデント(d)を基準に、先行する行(b, c)のインデントを調整：
```
- b
  - c
- d
```

## 原因

`pasteNodesFromText` のminLevel正規化は最小値を0にするが、先頭行がlevel 0でない場合（例: b=1, c=2, d=0）、level 1のノードに対する親（level 0）が作成前に参照され、`baseParentId` にフォールバックしてツリーが壊れる。

## 対策

minLevel正規化後、先頭行がlevel 0でない場合にツリー構造正規化パスを追加：

```javascript
if (parsed[0].level > 0) {
    var cap = 0;
    for (var r = 0; r < parsed.length; r++) {
        if (parsed[r].level > cap) {
            parsed[r].level = cap;
        }
        cap = parsed[r].level + 1;
    }
}
```

- 先頭行 → level 0 に強制
- 以降の各行 → `前行のlevel + 1` 以下に制約
- 結果: b(1),c(2),d(0) → b(0),c(1),d(0)
