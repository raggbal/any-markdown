# v0.195.564 リリースノート

## 修正1: Notes Undo/Redo スコープ分離

### 課題
Notes editor (.note) で outliner + sidepanel markdown を開いた状態で Cmd+Z / Cmd+Shift+Z を実行すると、sidepanel markdown を編集中にもかかわらず、outliner 側でも undo/redo が同時に発火してしまう。

### 原因
キーボードイベントの伝播順序で、Cmd+Z が2つの異なるハンドラで処理されていた:

1. **editor.js の capture-phase ハンドラ**（line 5977, `addEventListener(..., true)`）が先に発火し、`EditorInstance.getActiveInstance()` で sidepanel の `_undo()` を正しく呼ぶ
2. **outliner.js のグローバルハンドラ**（line 3833, bubble phase）が続いて発火。`document.activeElement` が `.outliner-text` でなければ outliner の `undo()` も実行する。sidepanel editor 内にフォーカスがある場合、`activeElement` は `.editor` 要素（`.outliner-text` ではない）のため、outliner の undo も発火してしまう

### 対策
outliner.js のグローバルキーハンドラの undo/redo 部分にガード条件を追加:

```javascript
if (sidePanelInstance && sidePanelEl && sidePanelEl.contains(document.activeElement)) { return; }
```

- `sidePanelInstance` が存在する（sidepanel が作成されている）
- `sidePanelEl` が存在する（DOM要素がある）
- `sidePanelEl.contains(document.activeElement)` が true（フォーカスが sidepanel 内にある）

この3条件すべてが true の場合のみ、outliner の undo/redo をスキップする。検索バーフォーカス時は `.side-panel` の外にあるためガード不発動、従来通り outliner undo/redo が動作する。

### 変更ファイル
- `src/webview/outliner.js` — グローバルキーハンドラに sidepanel ガード追加（undo, redo の2箇所）

## 修正2: Scope In 検索インジケーター

### 課題
Outliner で scope in している状態で検索を行うと、検索はスコープ内に限定されるが、UIとしてそれが明示されていない。ユーザーが「全体を検索している」と誤認する可能性がある。

### 対策
検索バーの直上に `.outliner-scope-search-indicator` 要素を追加し、scope in 中に「Search in scope」（i18n対応）のタグ/バッジを表示する。

**DOM構造**: `.outliner-search-bar` の直前に兄弟要素として追加。既存レイアウトに影響なし。

**表示制御**: `updateScopeSearchIndicator()` 関数を新規追加し、`setScope()` から呼び出す。`currentScope.type === 'subtree'` なら表示、それ以外は非表示。

**i18n**: 7言語対応（en: "Search in scope", ja: "スコープ内を検索" 等）

### 変更ファイル
- `src/outlinerWebviewContent.ts` — indicator DOM 追加
- `src/notesWebviewContent.ts` — 同上
- `src/webview/outliner.js` — `updateScopeSearchIndicator()` 関数追加、DOM参照取得、`setScope()` から呼び出し
- `src/webview/outliner.css` — indicator スタイル追加
- `src/i18n/locales/*.ts` — `outlinerSearchInScope` キー追加（7言語）
- `test/build-standalone-outliner.js` — テスト用HTML更新
- `test/build-standalone-notes.js` — テスト用HTML更新

## 修正3: Outliner → Sidepanel Markdown へのリスト形式ペースト

### 課題
Outliner で複数ノードを選択して Cmd+C し、sidepanel markdown editor で Cmd+V すると、プレーンテキスト（タブインデント）として貼り付けられ、Markdownリスト形式にならない。

### 対策
outliner.js の Cmd+C / Cmd+X ハンドラで、複数ノード選択時に `navigator.clipboard.write()` を使用し、`text/plain` に加えて `text/html` としてネストされた `<ul>/<li>` 構造もクリップボードに書き込むようにした。

**HTML生成**: `buildSelectedNodesHtml(nodesData)` 関数を新規追加。`getSelectedNodesData()` の結果（`[{text, level}]`）からネストされた `<ul>/<li>` 構造を生成。

**Markdown変換**: sidepanel markdown editor の既存 paste ハンドラが `text/html` → Turndown でMarkdownリスト（`- item` + 2sp indent）に自動変換する。editor.js の変更は不要。

**フォールバック**: `clipboard.write()` が失敗した場合は `writeText()` にフォールバック。

**内部クリップボード整合性**: `internalClipboard` の照合は `text/plain` ベースのため影響なし。

### 変更ファイル
- `src/webview/outliner.js` — `buildSelectedNodesHtml()`, `writeClipboardWithHtml()` 関数追加、Cmd+C / Cmd+X ハンドラ変更
