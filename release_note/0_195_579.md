# v0.195.579 リリースノート

## .md ファイルの Outliner 取り込み機能（ファイルピッカー方式）

### 課題

Notion / Obsidian 等から出力した `.md` ファイルを Outliner エディタに取り込む手段がなかった。

### 対策

#### 設計

⋮メニューに「Import .md files...」を追加。クリックでファイルピッカーダイアログを表示し、選択した `.md` ファイルをページノードとして取り込む。

- VSCode / Electron 両環境で同じ挙動（ファイルピッカーダイアログ方式に統一）
- Standalone Outliner / Notes Editor 両対応

#### 実装

**新規ファイル:**
- `src/shared/markdown-import.ts` — 共通ロジック
  - `importMdFile()` / `importMdFiles()`: ファイル読み込み→H1抽出→変換→画像コピー→ページ保存
  - `normalizeMarkdownPlainText()`: テーブル正規化（editor-utils.js と同等ロジック）
  - `normalizeMarkdownFromHtml()`: 将来のHTML取り込み用スタブ
  - `processImages()`: 画像参照解析→コピー（リネーム）→パス書き換え。URLエンコードされたパスも `decodeURIComponent` で対応（Notion対応）
  - `extractH1Title()`: 最初のH1テキスト抽出

**変更ファイル:**
- `src/webview/outliner.js`: ⋮メニューに「Import .md files...」追加。`importMdFilesResult` メッセージ受信でノード作成
- `src/shared/outliner-host-bridge.js`: `importMdFilesDialog()` メソッド追加
- `src/shared/notes-host-bridge.js`: `importMdFilesDialog()` メソッド追加
- `src/outlinerProvider.ts`: `importMdFilesDialog` ハンドラ（`showOpenDialog` + `importMdFiles`）
- `src/shared/notes-message-handler.ts`: `importMdFilesDialog` ハンドラ + `NotesPlatformActions` 拡張
- `src/notesEditorProvider.ts`: `importMdFilesDialog` platform action 実装

**機能詳細:**
- H1 があればノードtext、なければ "Untitled"
- 複数ファイル同時選択対応（ファイル名順で挿入）
- フォーカス中のノードの後に兄弟として挿入
- 画像: 元ファイルからの相対パスで解決→コピー（`image_{timestamp}_{random}.{ext}`）→ページからの相対パスに書き換え
- URLエンコードされた画像パス（Notion等）を `decodeURIComponent` でデコード
- URL画像（http/https）や存在しないファイルはスキップ

## Turndownインラインエスケープ除去 (v0.195.574)

### 課題

リッチテキストソース（Notion等）からMarkdownをペーストすると、`## 1\. サービス概要` のように不要なバックスラッシュが挿入された。

### 対策

Turndown変換後にインラインエスケープ（`\*`, `\_`, `` \` ``, `\[`, `\]`, `\\`, `\.`）を除去する処理を追加。
