# v0.195.554 リリースノート

## Outliner: rootIdsフォールバックコードの削除

### 課題

v0.195.553で `_ensureChildren()` のchildren再構築ロジックを削除したが、`deserialize()` 内に残っていた「rootIdsが空の場合にparentId===nullのノードをrootに追加する」フォールバックコードが不要になっていた。

### 原因

このフォールバックは旧形式データ（rootIdsが保存されていない時代）のための互換コードだが、現行フォーマットでは `serialize()` が必ず `rootIds` を含めて保存するため到達しない。不要コードの残存。

### 対策

#### 設計

- `deserialize()` 内のrootIdsフォールバックループを削除
- コメントを簡潔化（「各ノードの children が未設定の場合のフォールバック」）

#### 実装

`src/webview/outliner-model.js` の `deserialize()` から以下を削除:
- `rootIds.length === 0` 時に `parentId === null` のノードを探してrootIdsに追加するループ（9行）
