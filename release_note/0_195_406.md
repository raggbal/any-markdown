# v0.195.406 - データベースAction Panel（ページ追加機能）

## 課題

データベースエディタ（.mdd）の「+ New Page」ボタンは、pagesフォルダに固定ファイル名（pageId.md）で即座にページを作成する単一動作のみだった。以下の操作ができなかった：

1. 既存の.mdファイルをデータベースに追加する
2. 任意のパスにmdファイルを新規作成して追加する
3. ページのリンク名（タイトル）を作成時に指定する

## 対策

### 設計

「+ New Page」ボタンとCtrl+Nに、Notionライクなフローティング**Action Panel**を導入。3ステップのフローで操作する：

1. **メニューステップ**: 2つのアクションから選択
   - 「ページ追加」— pagesフォルダにtimestamp形式のmdを自動作成
   - 「パス指定でページ追加」— パス入力でファイルを指定
2. **パス入力ステップ**（パス指定時のみ）: オートコンプリート付きパス入力
3. **リンク名確定ステップ**（共通）: デフォルト=パス、全選択状態でEnter確定

### 実装

#### 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `out/shared/database-host-bridge.js` | `searchFiles(query)`, `createPageAtPath(relativePath, content)` メソッド追加 |
| `out/databaseProvider.js` | `searchFiles`（`vscode.workspace.findFiles`で.md検索、最大10件）、`createPageAtPath`（指定パスにmd作成、中間ディレクトリ自動作成）ハンドラ追加 |
| `out/webview/database-editor.js` | Action Panel全体（showActionPanel, showPathInput, showLinkNameConfirm, finalizeAddPage等）、メニュー↑↓+Enter、パス入力オートコンプリート（300msデバウンス、↑↓選択）、リンク名確定（全選択状態）、Escapeで前ステップに戻る、fileSearchResultsメッセージハンドラ |
| `out/webview/database-styles.css` | `.db-action-panel-*` スタイル一式（パネル、メニューアイテム、パス入力、オートコンプリートリスト、リンク名確定） |

#### ファイル名形式

画像ペースト（Cmd+V）と同じ `Date.now()` タイムスタンプ形式：
- 通常: `1710329456789.md`
- 衝突時: 同タイムスタンプが存在する可能性は極めて低いが、上位でgenerateIdがpageId生成

#### メッセージフロー

```
Webview → VSCode:
  searchFiles { query } → fileSearchResults { results[], query }
  createPageAtPath { relativePath, content } → pageCreatedAtPath { relativePath, absolutePath }

キーボード操作:
  メニュー: ↑↓選択、Enter実行、Escape閉じる
  パス入力: 入力→300msデバウンス→searchFiles、↑↓候補選択、Enter確定、Escape→メニューに戻る
  リンク名: Enter確定、Escape→メニューに戻る
```
