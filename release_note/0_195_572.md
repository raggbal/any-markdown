# v0.195.572 リリースノート

## 1. Markdown: セル内改行を含むテーブルの貼り付け対応

### 課題
Markdownテーブルをプレーンテキストとして貼り付ける際、セル内に生の改行（`\n`）が含まれていると、`markdownToHtmlFragment()` の行単位パーサーがテーブル行を認識できず、テーブルが正しく生成されなかった。

### 原因
テーブル行の判定条件は「`|` で始まり `|` で終わる」だが、セル内改行があると1つの行が複数行に分割され、各行が条件を満たさなくなる。

### 対策
**設計**: ペーストハンドラのプレーンテキスト処理パスに前処理関数 `normalizeMultiLineTableCells()` を追加。`|` で始まるが `|` で終わらない行を「折れたテーブル行」と判定し、後続行を `<br>` で結合して1行に正規化する。

**実装**:
- `editor-utils.js` に `normalizeMultiLineTableCells(text)` 関数を追加
- ペーストハンドラ（`editor.js`）でプレーンテキスト取得後に呼び出し
- 既存の `parseInline()` が `<br>` リテラルを実際の `<br>` 要素に変換する仕組みをそのまま活用
- 安全対策: 50行以内に閉じる `|` が見つからなければ結合を中止

## 2. Outliner: `saveBaseline()` 呼び出し漏れ修正

### 課題
Notes ファイル切替時に Undo スタックがクリアされず、Cmd+Z で前のファイルのデータに戻ってしまう。

### 原因
v0.195.564 で `saveBaseline()` 関数を新規追加した際、`init()` と `applyExternalUpdate()` は正しく置換されたが、`updateData` ハンドラ（Notes ファイル切替パス）の `saveSnapshot()` が `saveBaseline()` に変更されなかった（変更漏れ）。

### 対策
`outliner.js` L4102: `saveSnapshot()` → `saveBaseline()` に修正。

## 4. Outliner: スコープヘッダーのCSSクラス名統一

### 課題
テストが `.outliner-scope-header` クラスを期待しているが、実装は `.is-scope-header` を使用。設計書の記載は `.outliner-scope-header`。

## 5. Outliner: `toggleCollapse()` で子の数表示を追加

### 課題
`createNodeElement()` では折りたたみ時にバレットに子の数を表示するが、`toggleCollapse()` でクリック折りたたみした場合は子の数が表示されなかった。

### 原因
`toggleCollapse()` は `is-collapsed` クラスと `dataset.collapsed` を切り替えるだけで、`.outliner-child-count` span を追加/削除していなかった。

### 対策
`toggleCollapse()` 内で、collapsed 時に `outliner-child-count` span を追加、展開時に削除する処理を追加。

## 6. Outliner: `saveSnapshot()` の初回編集スキップ問題修正

### 課題
`saveBaseline()` 後に Enter で新ノード作成 → Cmd+Z しても元に戻らない。

### 原因
`saveSnapshot()` が Enter ハンドラでモデル変更前に呼ばれた時点で、`baselineSnapshot === snapshot` のため早期リターンし、undoStack に何も保存されなかった。undo() 時にデバウンスフラッシュで差分を検出しても、undoStack の top が current と同じ状態のため実質的に何も復元されなかった。

### 対策
`saveSnapshot()` の L102 の「baseline と同じなら保存しない」早期リターン条件を削除。baseline と同じでも undoStack に push するようにした。これにより Enter/Tab/Backspace 等の構造変更ハンドラが呼ぶ `saveSnapshot()` が確実に pre-edit 状態を記録する。
