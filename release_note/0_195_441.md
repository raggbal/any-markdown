# v0.195.441 - アウトライナー機能追加 (.mmd ファイル対応)

## 課題

WorkFlowy/Dynalist風のアウトライナー機能を fractal に追加したい。`.mmd` ファイルをツリー構造で編集でき、ノードをページ化すると `pages/{pageId}.md` ファイルを作成し、fractal の WYSIWYG エディタで編集できるようにしたい。

## 原因（新機能のため該当なし）

新規機能の実装。

## 対策

### 設計

- `.mmd` ファイル用の `CustomTextEditorProvider`（OutlinerProvider）を新設
- JSONベースのデータモデル（OutlinerModel）でツリー構造のCRUD操作を管理
- Dynalist互換の検索エンジン（OutlinerSearch）でAND/OR/NOT/タグ/フレーズ/演算子をサポート
- ページ機能: ノードに `@page` と入力（Space/Enter確定時）するとページ化。`pages/` フォルダに UUID v4 の pageId で `.md` ファイルを自動作成
- ページを開くと `vscode.openWith` で fractal エディタが `ViewColumn.Beside` に表示
- webview-host間通信は `outliner-host-bridge.js`（HostBridgeパターン踏襲）
- JSONコンテンツはBase64エンコードでwebviewに渡す（XSS防止）

### 実装（新規ファイル）

| ファイル | 役割 |
|---------|------|
| `src/outlinerProvider.ts` | CustomTextEditorProvider。JSON読み書き、ページファイル管理、外部変更検知 |
| `src/outlinerWebviewContent.ts` | webview HTML生成。CSP、Base64コンテンツ注入、スクリプトインライン化 |
| `src/shared/outliner-host-bridge.js` | VSCode用HostBridge（syncData, save, makePage, openPage, openLink等） |
| `src/webview/outliner-model.js` | データモデル。ツリーCRUD、タグ解析、インデント/アウトデント、シリアライズ |
| `src/webview/outliner-search.js` | 検索エンジン。トークナイズ→AST構築→マッチング。演算子: `in:title`, `has:children`, `is:page`, `is:task` |
| `src/webview/outliner.js` | UI層。ツリー描画、インライン編集、キーハンドラ、Undo/Redo、コンテキストメニュー |
| `src/webview/outliner.css` | スタイル。ノードレイアウト、バレット、折りたたみ、タグ、検索バー |
