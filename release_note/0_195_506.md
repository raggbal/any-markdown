# v0.195.506 — Notes機能 3点修正

## 修正点1: 空フォルダ時にdefault outlinerを自動作成

### 課題
Notesフォルダとして空のフォルダを追加して開くと、アウトラインが1つも存在せず、エディタが空状態で表示される。Electron版では自動で「default」アウトラインが作成されていた。

### 原因
`notesEditorProvider.ts` の `openNotesFolder()` でファイル一覧が空の場合、そのまま空状態で WebviewPanel を開いていた。Electron版の `welcome-create-outliner-folder` ハンドラでは `createFile('default')` を呼んでいたが、VSCode Notes版にはこのロジックがなかった。

### 対策
`openNotesFolder()` でファイル一覧取得後、空の場合に `fileManager.createFile('default')` を呼び出してからファイル一覧を再取得するようにした。Electron版と同じタイミング・同じ挙動。

## 修正点2: Notes経由時にSet page directoryを無効化

### 課題
Notes機能では各アウトラインの `pageDir` はファイル作成時に自動設定される（`./outlineId`）。メニューの「Set page directory...」で手動変更すると整合性が崩れる可能性がある。

### 原因
`outliner.js` の `toggleMenuDropdown()` はモードを区別せず常に「Set page directory...」メニュー項目を表示していた。

### 対策
`outliner.js` で `.notes-layout` 要素の有無をDOM検査し、Notes mode の場合は「Set page directory...」メニュー項目を生成しないようにした。既存のoutlinerモード（.outファイル単体開き）には影響なし。

## 修正点3: パネル閉じ時のトグルボタン重なり修正

### 課題
Notlines左パネルを閉じると、パネル再表示ボタン（▶）が `position: absolute; left: 8px` で表示され、検索バーの検索モード切り替えトグルボタンと重なる。

### 原因
パネル再表示ボタンの表示時に検索バーの左パディングが変わらないため、ボタン同士が重なっていた。

### 対策
`notes-body-html.js` に CSS ルール `.notes-file-panel.collapsed ~ .notes-main-wrapper .outliner-search-bar { padding-left: 44px; }` を追加。パネルが折り畳まれた時だけ検索バーの左パディングを広げてトグルボタンとの重なりを回避。
