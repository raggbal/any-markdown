# v0.195.330 — ツールバースクロールボタンの背景透過修正

## 課題

ツールバーの `<` `>` スクロールナビゲーションボタンの背景が透明で、後ろのツールバーボタンが透けて見えてしまい、視認性が悪かった。

## 原因

v0.195.329 で `opacity: 0.8` → `opacity: 1` に修正したが、依然として透過していた。

VSCode Webview環境では、ブラウザのデフォルトスタイルシートに加え、VSCode独自のスタイルが `button` 要素に対して `background: transparent` を適用している。このため、CSS カスタムプロパティ `var(--toolbar-bg)` で背景色を指定しても、VSCode側のスタイルの詳細度（specificity）が競合し、背景が透明のままになっていた。

スタンドアロンHTMLテスト環境ではVSCode Webviewのスタイルが適用されないため、この問題は再現しなかった。

## 対策

### 設計

`.toolbar-scroll-btn` の `background` プロパティに `!important` を付与し、VSCode Webviewのデフォルトスタイルを確実にオーバーライドする。

### 実装

**`src/webview/styles.css`**:

```css
.toolbar-scroll-btn {
    background: var(--toolbar-bg, #f6f8fa) !important;
}
.toolbar-scroll-btn:hover {
    background: var(--selection-bg) !important;
}
```

- `!important` によりVSCode Webviewの `button { background: transparent }` を確実に上書き
- フォールバック値 `#f6f8fa` を追加し、CSS変数が未定義の場合にも不透明な背景を保証
- hover状態にも `!important` を追加し、一貫性を確保
