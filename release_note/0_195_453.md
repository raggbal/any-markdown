# v0.195.453 - Shift+↑↓選択が2ノードに限定されるバグ修正

## 課題

Shift+↑↓で範囲選択しても、常にカーソルのあるノード＋上か下の1ノードの2ノードしか選択されない。3ノード以上の連続選択ができない。

## 原因

`selectRange()` 内で `clearSelection()` を呼んでおり、`clearSelection()` が `selectionAnchorId = null` にリセットしていた。次のShift+矢印で `if (!selectionAnchorId)` が真になり、現在ノードが新しいアンカーにセットされるため、常に「現在ノード＋1」の2ノードしか選択できなかった。

## 対策

`clearSelectionVisual()` 関数を新設。DOMのハイライトクリアと `selectedNodeIds.clear()` のみ行い、`selectionAnchorId` はリセットしない。`selectRange()` 内ではこちらを使用するよう変更。
