# v0.195.502 — pageDir永続化 + メニューUI

## 課題

outファイルのJSON内に設定した `pageDir`（outファイル個別のページ保存ディレクトリ設定）が、outliner上で何か編集するたびに消失する。

### 症状
1. `setPageDir` でpageDirを設定 → outファイルのJSONに `pageDir` フィールドが書き込まれる
2. outliner上でノードを編集する
3. `syncToHostImmediate()` が `model.serialize()` の結果でJSON全体を上書き
4. `model.serialize()` は `version`, `rootIds`, `nodes` のみを返すため、`pageDir` が消失
5. 以降の `@page` 操作はデフォルトの `./pages` に保存される

### 影響
- 3階層の優先順位（default `./pages` < VSCode設定 `outlinerPageDir` < outファイル個別 `pageDir`）のうち、最上位の「outファイル個別設定」が事実上機能しない

## 原因

`searchFocusMode` は既に同パターンで永続化されていたが、`pageDir` には同様の処理が未実装だった。

具体的には以下の4箇所が欠落:
1. `init()` で初期データからの `pageDir` 読み込み
2. `syncToHostImmediate()` での `pageDir` 付加
3. `updateData` ハンドラでの外部変更時の `pageDir` 復元
4. `pageDirChanged` ハンドラでの設定変更の変数反映

## 対策

### 設計
`searchFocusMode` パターンに準拠。グローバル変数でwebview側にpageDirを保持し、syncData時にserialize結果に付加する。

### 実装（src/webview/outliner.js — 5箇所）

| 箇所 | 修正内容 |
|------|---------|
| L25 | `var pageDir = null;` グローバル変数追加 |
| L101-103 | `init()` で `data.pageDir` を復元 |
| L2295 | `syncToHostImmediate()` で `if (pageDir) { data.pageDir = pageDir; }` 追加 |
| L2303 | `updateData` ハンドラで `pageDir = msg.data.pageDir \|\| null;` 追加 |
| L2319 | `pageDirChanged` ハンドラで `pageDir = msg.pageDir \|\| null;` に修正 |

### メニューUI（v0.195.501で実装済み）
- 検索バー右端にメニューボタン（⋮）を配置
- ドロップダウンから「Set page directory...」を選択可能
- `outlinerWebviewContent.ts`, `outliner.css`, `outliner.js` に実装済み
