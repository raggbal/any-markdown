# 要件定義書 — Outliner editor 3点改善

> **対象**: `src/webview/outliner.js`, `src/webview/outliner.css`
> **バージョン**: v0.195.575

---

## 修正1: Markdownリンク記法のクリック対応 + URLペースト自動変換

### 背景

現在、outliner editorでは `[text](url)` のMarkdown記法がblur時（非編集時）に `<a>` タグとしてレンダリングされるが、contenteditableの中にあるためクリックしてもリンクを開けない。また、URLをペーストする際に手動でMarkdown記法を書く必要がある。

### 要件

| No | 要件 |
|---|---|
| L-1 | blur時（非編集時）に `renderInlineText()` で生成された `<a>` タグを通常クリックで外部ブラウザで開けるようにする |
| L-2 | クリックでリンクを開く対象は `[text](url)` 形式のMarkdownリンク記法のみ。プレーンURL（`https://...`）の自動検出は不要 |
| L-3 | URLをペースト（Cmd+V）した際、単一行かつURL形式（`http://` または `https://` で始まる）の場合、自動的に `[URL](URL)` 形式に変換してモデルに保存する |
| L-4 | リンクのクリック操作はHostBridge経由でホスト（VSCode/Electron）に委任し、外部ブラウザまたはVSCodeのリンクオープン機能で開く |
| L-5 | 編集モード（focus時）ではリンクはMarkdown記法のまま表示され、クリックでリンクを開く機能は無効 |

### 対象外

- プレーンURLの自動検出・自動リンク化
- Cmd+クリック等の修飾キー付きクリック（通常クリックで開く）

## 修正2: 複数ノード選択時のTab/Shift+Tabインデント・デインデント

### 背景

現在、Tab/Shift+Tabによるインデント・デインデントは単一ノード（フォーカスノード）のみに対応している。複数ノードをShift+クリックやShift+矢印キーで選択した状態でTab/Shift+Tabを押しても、フォーカスノードだけが処理される。

### 要件

| No | 要件 |
|---|---|
| I-1 | 複数ノードが選択されている状態（`selectedNodeIds.size > 0`）でTabを押した場合、選択されている全ノードをインデントする |
| I-2 | 複数ノードが選択されている状態でShift+Tabを押した場合、選択されている全ノードをデインデントする |
| I-3 | 処理はDOM表示順（上から下）で実行する。インデントは上から順、デインデントも上から順に処理する |
| I-4 | スコープ境界制限を遵守する：スコープルートの直接の子ノードはデインデント不可（既存のSI-7ルール） |
| I-5 | 処理後も選択状態を維持する |
| I-6 | 処理前にスナップショットを保存し、Undo/Redoに対応する |

### 対象外

- 単一ノード（選択なし）のTab/Shift+Tab動作の変更（既存動作を維持）

## 修正3: ペースト時の空行ノード作成抑制

### 背景

現在、テキストをCmd+Vでペーストすると、改行ごとにノードが作成される。中間の空行も空テキストのノードとして作成されるが、これは不要。最終行の空行のみスキップされている。

### 要件

| No | 要件 |
|---|---|
| P-1 | 複数行テキストのペースト時、空行（インデント除去後のテキストが空文字列の行）はノードを作成せずスキップする |
| P-2 | 最終行の空行スキップ（既存動作）は維持する |
| P-3 | 内部クリップボード（`clipboardNodes`）経由のペーストでは、空行スキップにより `clipNodeIndexMap` のマッピングが正しく維持されること |

### 対象外

- 単一行ペーストの動作変更
- スペースのみの行（`"   "`等）の扱い変更（既存動作を維持）
