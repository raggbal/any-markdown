# 要件定義書 — Outliner追加修正 v0.195.559

---

## 修正1: ノードのドラッグ&ドロップ移動

### 現状
ノードの移動はキーボードのみ（Cmd+Shift+↑/↓でスワップ、Tab/Shift+Tabでインデント/アウトデント）。マウスでのD&D移動は未実装。

### 要件

| No | 要件 |
|---|---|
| DND-1 | ノードをマウスでドラッグして別の位置にドロップで移動できる |
| DND-2 | 親ノードをD&Dする場合、子ノードもすべて一緒に移動する（サブツリーごと移動） |
| DND-3 | ドラッグ中のドロップ位置にインジケーター（青い線）を表示する |
| DND-4 | ドロップ位置は「ノードの上（兄弟として前に挿入）」「ノードの下（兄弟として後に挿入）」「ノードの子として挿入」の3パターンを判定する |
| DND-5 | 自分自身やその子孫ノードへのドロップは禁止する（循環参照防止） |
| DND-6 | ドラッグ元ノードは半透明（opacity: 0.4）で表示する |
| DND-7 | D&D後にundo/redoが効くこと（saveSnapshot連携） |
| DND-8 | D&D後にホスト同期（scheduleSyncToHost）すること |
| DND-9 | 折りたたまれたノードもD&D可能。折りたたみ内の子もサブツリーごと移動する |
| DND-10 | ドラッグはバレット（●）のドラッグで開始する。テキスト部分のドラッグはブラウザのテキスト選択を優先する |

## 修正2: サイドパネルMarkdownエディタの自動フォーカス

### 現状
サイドパネルを開いても、エディタにカーソルが入らず、ユーザーがクリックしないと編集を開始できない。

### 要件

| No | 要件 |
|---|---|
| AF-1 | サイドパネルのMarkdownエディタを開いたら、自動でエディタにフォーカスしカーソルを表示する |
| AF-2 | フォーカスはエディタの先頭（最初のブロック要素の先頭）に設定する |
| AF-3 | アニメーション完了後にフォーカスを設定する（DOM描画完了を待つ） |
